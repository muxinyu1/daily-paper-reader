## 摘要
AI编程智能体（如Claude Code和Gemini CLI）越来越多地通过第三方技能扩展自身：这些技能是打包了自然语言指令、可执行脚本和工具权限的markdown包。由于技能同时包含代码和面向智能体的指令，它引入了一种供应链依赖，其风险既非纯代码也非纯提示。检测工具从未针对涵盖这种混合空间的已验证真值进行过评估，导致其有效性未知，且仅基于野外数据的评估存在偏差。我们提出了MalSkillBench，这是首个经过运行时验证的恶意智能体技能基准测试：包含3,944个恶意技能，标注了一个108个单元的三维分类体系。其中3,214个来自一个闭环的“生成-验证-反馈”流水线，仅接受那些在Docker沙箱中、在系统调用监控和LLM评判下恶意行为被触发的样本；我们补充了703个野外样本和4,000个匹配的良性技能。我们的测量结果一致：代码注入的验证成功率达到94.5%，但提示注入仅为75.8%，这种脆弱性后来也使其难以被检测；野外样本范围狭窄，主要由一次加密货币盗窃活动主导（86.6%为单一行为，81%来自两个账户），但有一个虽小却架构新颖的尾部攻击了智能体控制平面；最强的技能专用检测器在代码注入上召回率达98.4%，但在提示注入和智能体控制攻击上表现崩溃，仅基于野外数据的评分会使排名波动高达66个召回点；供应链扫描器和提示注入防御各自只见技能的一半，且任何组合都无法恢复代码与指令之间的关系。因此，检测恶意技能需要同时对任务意图、代码和指令进行联合推理。我们发布了数据集、流水线、基线和结果。

## Abstract
AI coding agents such as Claude Code and Gemini CLI increasingly extend themselves with third-party skills: markdown packages bundling natural-language instructions, executable scripts, and tool permissions. Because a skill is at once code and agent-facing instruction, it introduces a supply chain dependency whose risk is neither pure code nor pure prompt. Detection tools have never been measured against verified ground truth spanning this hybrid space, leaving their effectiveness unknown and wild-only evaluations biased.   We present MalSkillBench, the first runtime-verified benchmark of malicious agent skills: 3,944 malicious skills labeled along a three-dimensional taxonomy of 108 cells. Of these, 3,214 come from a closed-loop Generate-Verify-Feedback pipeline admitting only samples whose malicious behavior fires inside a Docker sandbox under system-call monitoring and an LLM judge; we add 703 in-the-wild and 4,000 matched benign skills. Our measurements are consistent: code injection reaches 94.5% verification yield but prompt injection only 75.8%, the same fragility that later makes it hard to detect; the wild sample is narrow, dominated by one cryptocurrency-theft campaign (86.6% one behavior, 81% from two accounts) with a small but architecturally new tail attacking the agent control plane; the strongest skill-specific detector reaches 98.4% recall on code injection yet collapses on prompt-injection and agent-control attacks, and wild-only scoring swings the ranking by up to 66 recall points; supply-chain scanners and prompt-injection defenses each see only half of a skill, and no combination recovers the code-instruction relationship. Detecting malicious skills therefore requires reasoning jointly over task intent, code, and instructions. We release the dataset, pipeline, baselines, and results.

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：AI编码智能体（如Claude Code、Gemini CLI）通过第三方“技能”（skill）扩展能力。技能是一个SKILL.md包，包含自然语言指令、可执行脚本和工具权限。技能同时具备代码和面向智能体的指令，形成一种混合的软件供应链依赖，风险既非纯代码也非纯提示。
- **问题**：现有检测工具从未在覆盖这种混合空间的已验证真值上进行过评估，其有效性未知；仅基于野外（in-the-wild）数据的评估存在系统性偏差（因为野外样本分布极其狭窄）。
- **目标**：构建首个经过运行时验证的恶意技能基准测试，系统评估现有检测工具能力，揭示检测盲区。

## 2. 方法论：核心思想、关键技术细节、流程
- **核心思想**：通过闭环“生成-验证-反馈”流水线，从真实攻击模式中迁移知识，自动生成且运行时验证的恶意技能样本，形成全面覆盖的三维分类体系（攻击向量×恶意行为×插入策略，共计108个单元格）。
- **关键技术细节**：
  - **三维攻击空间**：
    - 攻击向量：代码注入（CI）、提示注入（PI）、混合攻击（MIXED）。
    - 恶意行为：15类（B1-B9为代码层行为，B10-B15为智能体推理层行为，PI专用）。
    - 插入策略：CI有4种（新脚本文件、函数追加、函数注入、内联代码块）；PI有3种（全伪装、部分注入、隐写）；MIXED有3种（下载+执行、配置+加载、获取+运行）。
  - **知识库**：CI知识库来自IntelliGraph（3,026个已确认恶意的PyPI包）；PI知识库来自四个提示注入语料库（WildJailbreak等，共20,961个载荷）。用LLM重新标注使其对齐到分类体系。
  - **生成代理**：使用Qwen3.5-35B（abliterated变体，去除拒绝方向）在本地通过Ollama服务。为每个目标单元格检索相关攻击示例，从3,458个良性技能模板池中随机选取一个模板作为伪装，然后生成完整的技能包（含SKILL.md、脚本、expected.json）。
  - **验证代理**：将候选技能部署在OpenCode智能体环境中（Docker容器，带strace和inotifywait监控），触发后收集系统调用跟踪和智能体文本输出。两层验证：
    - 层1：基于规则的IOC匹配（确定性）。
    - 层2：LLM语义分析（GPT-5.4-mini，阈值θ=0.7）。
    - 通过条件：至少一个预期行为被层1匹配或层2确认。
  - **反馈**：拒绝时，验证代理生成反馈（含原因、观察到的行为子集、建议），附加到生成提示中，生成代理最多重试3次。
- **算法流程**：遍历108个单元格，对每个单元格生成k_s个样本。每次：检索知识→采样模板→LLM合成→验证→通过则加入数据集，否则反馈重试，最多3次。最终共生成3,214个验证通过的样本。

## 6. 主要结论与发现
1. **攻击可实现性不均衡**：代码注入（CI）验证成功率94.5%，混合攻击（MIXED）91.9%，提示注入（PI）仅75.8%。PI中隐写插入策略最低（62.5%），全伪装最高（89.7%）。PI的可实现性低与其后检测难度高一致。
2. **野外攻击表面狭窄**：703个野外样本中，86.3%通过伪造依赖伪装传递，86.6%属于恶意软件投递（B4），81%来自两个账户，主要是加密货币盗窃活动。只有1.7%攻击智能体控制平面（会话生命周期、身份重写、指令层级覆盖），代表新型威胁。
3. **技能专用检测器性能参差不齐**：最强的Sentry Full达到98.4%召回率，但对PI攻击（特别是内容操纵B15）大幅下降。AI-Infra-Guard最平衡（F1 85.6%）。VirusTotal等签名工具召回率低（21.6%）。仅用野外子集评估导致排名剧烈波动（VirusTotal召回跳升66点，从垫底到领先）。
4. **工具迁移性差**：供应链扫描器和提示注入防御各自只能看到技能的一半。高召回转移工具产生大量误报（如OSSGadget FP 3,587/4,000）；高精度转移工具漏报严重。OR组合继承误报，AND组合使召回降到6%以下。简单组合无法恢复代码与指令之间的关系。
5. **检测需联合推理**：技能的恶意性通常存在于各部分之间的关系（所请求的权限/动作是否被广告任务证明合理）。现有单一领域工具和简单组合均无法实现这种关系推理。

## 7. 优点
- **首创性**：首个经过运行时验证的恶意技能基准，填补了该领域公开真值数据的空白。
- **方法论严谨性**：闭环生成-验证-反馈流水线确保了样本的真实恶意行为可触发；系统调用级证据+LLM语义分析两层验证提供了可靠标签。
- **覆盖全面**：三维分类体系108个单元格全部覆盖，且从真实攻击知识库迁移模式，避免了野外样本偏倚。
- **实验设计系统**：评估了12种检测器（含多种范式），对比了完整基准与仅野外子集的差异，暴露了评估偏差；同时评估了供应链/提示注入工具的迁移性，揭示了结构性限制。
- **开源性**：公开了数据集、代码、基线和评估流水线，促进可复现研究。
- **实际指导意义**：结论直接指导未来恶意技能检测的应去方向——联合推理任务意图、代码和指令。

## 8. 不足与局限
1. **野生样本覆盖不足**：尽管收集了703个野外样本，但分布极不均匀，主要局限在依赖伪装攻击。生成的3,214个样本虽覆盖全空间，但其现实性可能弱于真实野外样本（尽管通过知识迁移减轻）。
2. **良性样本集偏差**：良性集仅来自ClawHub高下载量技能，可能不包括低质量或边界良性技能，导致FP率估计偏低。
3. **验证代理依赖LLM**：PI行为的验证主要依赖GPT-5.4-mini的语义判断（阈值0.7），存在误判风险。虽然层1确定性证据用于CI，但PI层2判断的可靠性未经独立验证（论文用300个抽样人工审核确认，但未给出量化一致性指标）。
4. **生成代理的局限**：使用abliterated模型（去除对齐），虽然便于生成恶意内容，但其生成模式可能与真实攻击者存在差异（如风格、工具链、触发条件）。反馈机制可能引入循环偏差（验证失败后重试可能趋向于更“容易验证”的攻击）。
5. **基线限制**：评估仅限于工具发布时的版本，且部分工具使用不同LLM后端（GPT-5.4 vs GPT-4o vs Claude），比较公平性受影响。未考虑最新或未公开的检测器。
6. **算力和资源信息不透明**：未报告生成和验证过程的GPU数量、内存、时间等，不利于复现和规模评估。
7. **未覆盖的威胁**：论文聚焦于恶意技能检测，但未考虑其他类型的智能体攻击（如工具调用链、多步协作攻击）或防御（如运行时监控、权限分离）。攻击表面可能随生态系统发展而变化。
