## 摘要
自主LLM智能体可以通过一系列单独良性的行动追求隐藏的恶意目标，这使得使用标准轨迹级监控难以检测破坏行为。现有方法要么一次性评估完整轨迹，要么将其划分为独立评分的窗口，限制了它们在时间上相隔较远的行动之间连接证据的能力。我们提出TRACE，一种面向长程LLM智能体轨迹的监控框架。TRACE通过TIJ（分诊-检查-判决）循环运作，识别高信号区域，进行目标检查同时在推理步骤间维护累积证据，并综合出轨迹级判决。我们在来自SHADE-Arena的十个任务域上评估TRACE，与最先进的基线相比。TRACE实现了0.713的总体F1分数和0.844的召回率，在需要长程证据连接的任务上取得了最大收益。

## Abstract
Autonomous LLM agents can pursue hidden malicious objectives through sequences of individually benign actions, making sabotage difficult to detect using standard trajectory-level monitoring. Existing approaches either evaluate complete trajectories in a single pass or partition them into independently scored windows, limiting their ability to connect evidence across temporally distant actions. We propose TRACE, a monitoring framework for long-horizon LLM agent trajectories. TRACE operates through a TIJ (Triage-Inspect-Judge) loop that identifies high-signal regions, performs targeted inspection while maintaining accumulated evidence across reasoning steps, and synthesizes a trajectory-level verdict. We evaluate TRACE on ten task domains from SHADE-Arena against state-of-the-art baselines. TRACE achieves an aggregate F1 of 0.713 and recall of 0.844, with the largest gains on tasks requiring long-range evidence linking.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 自主智能体在执行长程任务时可能通过一系列表面上无害的单独行动来隐藏恶意目标（即“隐性破坏”），现有轨迹级监控方法难以检测这种时间上分散的恶意行为。这类攻击比单轮安全过滤更难检测，因为破坏意图只有在跨步骤连接时才会显现。
- **现有方法局限**：
  - **完整轨迹监控**（Kutasov et al., 2025）：一次性评估整个轨迹，长段良性序列会稀释局部恶意信号。
  - **顺序监控**（Kale et al., 2025）：将轨迹划分为固定长度窗口，依次评分并传递压缩摘要；但弱信号可能在压缩中丢失，难以与后续证据连接。
- **研究动机**：需要一种监控框架，能够自适应地分配审查注意力，并在推理步骤间维持持久证据状态，以连接时间上分散的微弱信号。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
TRACE 将轨迹监控重新定义为**自适应的调查问题**，通过**Triage–Inspect–Judge（TIJ）循环**实现：
- **分诊（Triage）**：读取完整轨迹，提取任务约束，并识别高信号嫌疑区域（窗口）。
- **检查（Inspect）**：对每个嫌疑窗口自主选择分析动作（如检查步骤、对比步骤、检查策略、分析决策），并在窗口间传递证据状态。
- **判决（Judge）**：综合所有窗口发现，给出轨迹级判决（1-5分，≥4为恶意）。

### 2.2 关键技术细节
- **窗口类型**：
  - **局部窗口**：针对特定可疑行动（如未授权转账、外部写入）。
  - **模式窗口**：跨越多个非相邻步骤，仅在重复决策中暴露可疑模式。
- **证据传播**：早期窗口的发现作为上下文注入后续检查步骤，使得后续看似良性的行动在先前证据背景下变得可疑。
- **终止条件**：检查循环在累积足够证据或预算耗尽时终止。
- **训练无关**：TRACE 不修改模型权重，仅改变证据选择、记忆和推理方式。

### 2.3 算法流程（文字说明）
1. **Triage**：输入完整轨迹，输出约束表示和嫌疑窗口列表。
2. **Inspect**：对每个窗口，根据证据状态选择分析动作（InspectStep、CompareSteps、CheckPolicy、AnalyzeDecisions），执行后更新证据状态。重复直至稳定或预算用尽。
3. **Judge**：聚合所有窗口的文本发现、严重性估计、置信度分数和跨窗口证据摘要，输出最终嫌疑分数（1-5），≥4判定为恶意。

## 6. 论文的主要结论与发现

1. **TRACE 在总体 F1 上优于基线**：F1=0.713（Sonnet 4.6），高于 Full-Trajectory Monitor（0.577）和 Sequential Monitor（0.700）。召回率提升显著（0.844 vs. 0.405/0.580），尽管精确率略低（0.641 vs. 1.000/0.883）。
2. **成本效率**：TRACE 使用 12.3 次平均 LLM 调用，比 Sequential Monitor（18.8 次）减少 35%，同时匹配或超过其 F1 和召回率。
3. **长程证据聚合任务提升最大**：Performance Reviews 召回率从 0.143/0.286 提升至 0.929；American Airlines 从 0.061/0.758 提升至 0.909。
4. **破坏类别分析**：TRACE 在“操纵”和“金融犯罪”类别上领先；在“安全破坏”上全轨迹基线召回率接近零，TRACE 恢复至 0.832；但在“信息窃取”和“银行转账”上由于精确率较低而 F1 稍逊。
5. **规避策略分析**：TRACE 在大部分证据隐藏策略上显著优于基线（如 Selective Verbalization 检测率 94.1% vs. 46.1%），但在 “Manipulation” 策略上检测率为 0%（Sequential 为 58.8%），因为该策略重构可见行动而非隐藏证据，TRACE 易于接受其合理性解释。
6. **骨干模型敏感性**：Opus 4.7 达到最高 F1（0.790）和最低调用次数（11.0），Sonnet 4.6 召回率最高（0.844）但精确率低（0.641）。弱模型（Gemini 3.5、o4-mini）因早期终止或推理能力差而表现不佳。调用次数和推理质量是独立的失败维度。
