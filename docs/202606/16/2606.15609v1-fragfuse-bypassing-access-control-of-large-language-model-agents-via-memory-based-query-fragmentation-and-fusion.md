## 摘要
大型语言模型（LLM）代理日益依赖长期记忆来支持复杂任务执行、用户个性化和领域适应。同时，人们正在探索针对LLM代理的新兴访问控制机制，以阻止违反策略的请求并防止滥用。我们揭示了由代理记忆操作引发的新型攻击面：本应触发访问控制的禁止内容可以被跨交互碎片化，以看似良性的形式存储于长期记忆中，随后通过记忆检索重建，而无需在最终用户查询中显式出现。我们提出FragFuse，这是首个利用长期记忆引入的时域通道使非特权用户能够绕过代理访问控制的攻击。FragFuse分三个阶段运行：（1）通过带片段掩膜的黑盒自适应查询识别拒绝响应片段；（2）使用标记载体查询将这些片段注入记忆；（3）通过后续攻击查询检索并融合存储的片段。尽管FragFuse可以手动为单个代理实例化，但我们进一步开发了一种基于替代模型的优化方案，用于调整融合指令和标记设计，从而在未违反攻击者威胁模型假设的情况下实现自动化攻击生成。我们在四种代表性代理设置和任务领域上评估了FragFuse，涵盖了三种最先进的代理访问控制机制。FragFuse在所有设置中实现了86.3%的平均绕过成功率和41.1%的平均端到端有害任务成功率，与无访问控制的配置相比，平均任务成功率仅降低4.4%。我们还表明，包括最先进的提示注入检测器和困惑度检测器在内的替代防御措施并不能有效应对这种攻击。

## Abstract
Large language model (LLM) agents increasingly rely on long-term memory to support complex task execution, user personalization, and domain adaptation. Meanwhile, emerging access-control mechanisms for LLM agents are being explored to block policy-violating requests and prevent misuse. We reveal a novel attack surface arising from agent memory operations: prohibited content that would trigger access control can be fragmented across interactions, stored in long-term memory in benign-appearing form, and later reconstructed through memory retrieval without appearing explicitly in the final user query. We propose FragFuse, the first attack that enables unprivileged users to bypass agent access control by exploiting this temporal channel introduced by long-term memory. FragFuse operates in three stages: (1) identifying rejection-responsive fragments via black-box adaptive querying with fragment masking; (2) injecting these fragments into memory using marker carrier queries; and (3) retrieving and fusing the stored fragments through a follow-up attack query. Although FragFuse can be instantiated manually for individual agents, we further develop a surrogate-based optimization scheme that tunes fusion instructions and marker designs, enabling automated attack generation without violating the attacker's threat-model assumptions. We evaluate FragFuse across four representative agent settings and task domains, covering three state-of-the-art agent access-control mechanisms. FragFuse achieves an average bypass success rate of 86.3% and an average end-to-end harmful task success rate of 41.1% across all settings, with only 4.4% average task-success degradation compared with configurations without access control. We also show that alternative defenses, including state-of-the-art prompt-injection detectors and perplexity detectors, do not effectively address this attack.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **研究背景**：大型语言模型（LLM）代理日益采用长期记忆机制来支持复杂任务、用户个性化和领域适应。同时，为了遵守政策、防止滥用并提高资源效率，代理开始配备访问控制（AC）机制，基于规则或LLM检测用户查询中的违禁内容。
- **核心问题**：现有访问控制仅检查当前用户查询，假设所有违禁意图都会显式出现在当前查询中。但长期记忆引入了**时域通道**：攻击者可以在早期交互中以良性形式将敏感信息写入记忆，然后在后续查询中通过记忆检索重建这些信息，从而绕过访问控制。
- **整体含义**：本文首次识别并利用这一攻击面，提出了 **FragFuse** 攻击，揭示了记忆机制不仅仅是存储组件，更可成为破坏访问控制的有效武器。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程
- **核心思想**：将原本会被AC拒绝的禁止查询 q 碎片化，通过三次与代理的正常交互，实现绕过和任务执行。
  - 第一次交互：注入包含敏感碎片的良性载体查询到记忆。
  - 第二次交互：通过攻击查询触发记忆检索，并利用融合指令将碎片拼回，诱导代理执行原禁止意图。
- **三阶段攻击流程**：
  1. **敏感片段发现 (Sensitive Fragment Discovery)**
     - **输入**：禁止查询 q、公开的AC策略规则。
     - **过程**：迭代执行片段提取与验证。使用LLM-based **FragExtractor** 从当前查询中提取一个可能的敏感片段 ft，将其替换为标记 vt（如角括号 `<>`），形成临时掩码查询 q(t+1)。提交给代理的AC，观察接受/拒绝信号。若拒绝，则继续下一次迭代，直至掩码查询被接受或达到最大迭代次数（通常3次）。
     - **输出**：敏感片段集合 {ft} 及最终掩码查询 qmask。
  2. **载体查询构建与注入 (Carrier Construction & Injection)**
     - **输入**：敏感片段 {ft}、标记 {vt}、一个良性主机查询 qhost（确定能通过AC）。
     - **过程**：将每个敏感片段按单词粒度拆解，并用标记包裹，追加到 qhost 后形成载体查询 qcar = [qhost, v1: f1, ..., vℓ: fℓ]。提交qcar给代理，该查询通过AC，其执行结果被写入记忆。
     - **输出**：存入记忆的载体记录。
  3. **片段检索与融合 (Fragment Retrieval & Fusion)**
     - **输入**：qmask、qhost、融合指令 Ifuse。
     - **过程**：构建攻击查询 qatt = {qhost, Ifuse, qmask}。qhost 保证基于相似度的记忆检索能找到载体记录；Ifuse 指导代理从载体记录中提取标记包裹的片段并填入qmask对应位置，重建出原禁止查询 q。
     - **输出**：代理执行融合后的禁止任务。
- **融合指令优化 (Surrogate Optimization)**：
  - **动机**：手动编写的 Ifuse 可能因过长、指令混乱等原因导致检索失败或融合失败。
  - **方法**：使用替代模型（GPT-4o）和替代查询融合任务，通过遗传算法优化 Ifuse。目标函数包含：
    - 检索损失 Lret：最小化 qatt 与 qhost 的语义距离。
    - 融合损失 Lfus：最大化替代模型生成的融合查询与 q 的语义相似度。
    - 流畅性约束 Lcoh ≤ ηcoh：确保 qatt 的困惑度（perplexity）与良性查询一致，以逃避困惑度检测。
  - **优化流程**：初始化种子指令 → 每一代通过LLM重写、交叉、变异生成候选 → 评估损失与约束 → 选择精英 → 迭代直至收敛。

## 6. 论文的主要结论与发现
- **FragFuse能高效绕过访问控制**：在24个配置中平均BSR达86.3%，平均E2E-SR达41.1%，而基线仅为5.7%。

- **查询效率高**：敏感片段发现平均仅需1.18-2.37次查询（表2），整个攻击仅需2-3次交互。
- **鲁棒性强**：对不同的记忆大小、检索方式、记忆格式、标记符选择等均有较好的适应性。
- **现有防御无效**：
  - 通用提示注入检测器（PromptArmor、PromptGuard）对FragFuse的分类效果不佳，尤其在InspAgent上。
  - Perplexity分布显示载体/攻击查询与良性查询高度重叠，难以区分。
- **事后检查虽为最后防线，但成本高且不实用**：即使能检测到违规动作，资源损耗已发生。
- **揭示了记忆作为新攻击表面的紧迫性**：后续需要针对记忆时域通道设计更安全的访问控制。

## 7. 优点：方法或实验设计上的亮点
- **首次系统定义并利用时域记忆通道**：提出了一个全新视角，将记忆视为可被攻击者主动操作以绕过AC的组件。
- **完全黑盒且无需特权**：攻击者仅为普通用户，无法访问记忆或AC内部逻辑，仅通过正常交互完成攻击，现实威胁高。
- **自动化程度高**：敏感片段发现和融合指令优化均可自动化，手动版本也能工作，实用性强。
- **严密且高效的优化框架**：针对黑盒场景设计了离线替代优化，兼顾攻击成功率和隐蔽性（困惑度约束）。
- **实验评估全面**：覆盖多个代理领域、多种AC机制、不同骨干模型，进行大量消融分析，结果统计可靠。
- **对防御的深入讨论**：不仅是攻击方法，还实证检验了现有通用防御的不足，为后续防御研究奠定基础。

## 8. 不足与局限：包括实验覆盖、偏差风险、应用限制
- **任务领域覆盖有限**：仅四个场景（电商、OS、Web导航、工具调用），未覆盖医学、科学发现、金融等其他重要领域。作者指出主要原因是这些领域缺乏可系统测试的访问控制策略。
- **依赖模型能力**：融合指令的执行需要代理的底层LLM具备足够强的推理和指令跟随能力。若模型较弱，攻击可能失败。
- **对事后动作检查的脆弱性**：如果部署了严格的事后动作检查（如ShieldAgent），即使在查询层面绕过，仍可能在动作执行时被拦截。论文认为事后检查成本高、通用性差，但并未完全否定其有效性。
