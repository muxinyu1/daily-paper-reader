## 摘要
代理型大语言模型（LLM）系统现在可以执行动作，而不仅仅是生成文本。当模型输出触发特权操作（如shell命令、浏览器自动化或外部工具调用）时，安全问题从仅对齐转变为系统配置和结构设计。我们分析了OpenClaw，一个自托管的多智能体系统，其中LLM输出可以执行命令并与工具和服务交互。我们测量了妥协概率、边界失败、特权漂移，以及这些指标随攻击者能力增强的变化。使用一个智能体时，妥协概率为0.24。使用七个智能体时，当系统执行某个动作，如果任何单个智能体提议该动作，妥协概率上升至0.86。模型本身未变；增加来自输出聚合。提示注入导致不稳定性在系统中传播。攻击面熵从0.42增加到0.71，表明利用路径的分布更广。平均特权漂移从0.03增加到0.21，表明意外的权限提升。正向升级曲率为0.08，表明随着攻击者能力增强，权限增长更快。防御控制（包括策略门控和执行过滤）将妥协概率降低0.10，边界失败降低0.10，特权漂移降低0.02，所有这些在p < 0.0001时具有统计显著性。系统仍然敏感，但缓解效果可测量。不同模型的注入缓解成功率不同：GPT-5.2为0.37，Llama-4-Maverick为0.35，DeepSeek-R1为0.31。当任何单个智能体都能触发执行时，最脆弱的智能体决定系统暴露程度。缓解措施略微降低了任务效用（从0.93降至0.89），并将中位延迟从420毫秒增加到468毫秒。

## Abstract
Agentic large language model (LLM) systems can now execute actions, not only produce text. When model outputs trigger privileged operations such as shell commands, browser automation, or external tool calls, the security problem shifts from alignment alone to system configuration and structural design. We analyze OpenClaw, a self-hosted multi-agent system in which LLM outputs can execute commands and interact with tools and services. We measure compromise probability, boundary failures, privilege drift, and how these metrics change as attacker capability increases. With one agent, the compromise probability is 0.24. With seven agents, when the system executes an action, the compromise rises to 0.86 if any single agent proposes it. The models do not change; the increase comes from output aggregation. Prompt injection propagates instability across the system. Attack surface entropy increases from 0.42 to 0.71, indicating a broader distribution of exploit paths. The mean privilege drift increases from 0.03 to 0.21, indicating unintended authority gain. Positive escalation curvature of 0.08 indicates that privilege grows faster as attacker capability increases. Defensive controls, including policy gating and execution filtering, reduce compromise probability by 0.10, boundary failures by 0.10, and privilege drift by 0.02, all statistically significant at p < 0.0001. The system remains sensitive, but the mitigation impact is measurable. Injection mitigation success differs across models: 0.37 for GPT-5.2, 0.35 for Llama-4-Maverick, and 0.31 for DeepSeek-R1. When execution can be triggered by any single agent, the most vulnerable agent determines system exposure. Mitigations slightly reduce task utility from 0.93 to 0.89 and increase median latency from 420 ms to 468 ms.

*   **核心问题：** 随着LLM系统从单纯文本生成进化到能够执行代码、调用工具和访问外部系统的“代理型AI”（Agentic AI），其安全范式发生了根本性转变。安全问题不再仅仅是“对齐”（Alienment）问题（即模型是否拒绝对抗性指令），更包含了系统架构与配置如何导致攻击面急剧扩张。本研究旨在回答：在多智能体系统中，当多个模型的输出被聚合以决定执行特权操作时，是如何改变整体攻击面、引入新的信任边界违规，并导致权限动态升级的？
*   **整体含义：** 论文以**OpenClaw**（一个自托管的开源多智能体LLM系统）为例，揭示了多智能体系统的风险并不仅仅等于各模型风险的叠加。由于“**任一代理提议即可执行**”这类宽松的**聚合策略**，系统的“**短板效应**”被放大，最脆弱的单个代理决定了整个系统的暴露程度。这强调了在设计安全的代理系统时，必须从个体模型安全转向系统架构层面的安全设计。

        *   **累计边界不稳定性 (CBI, Cumulative Boundary Instability)：** 量化攻击后信任边界（如输入-推理边界、推理-执行边界）的脆弱性程度。

        *   **反事实性缓解验证算法：** 在相同的攻击条件下，并行运行基线系统和经过特定缓解策略加固后的系统，通过对比两者的风险指标差异（ΔR，如Δ攻击面、ΔCBI）来量化缓解措施的真实因果效应。

6.  **聚合策略比模型本身更重要：** 采用“多数共识”规则能有效抑制多智能体带来的风险放大效应，将7个智能体的妥协概率从0.86降至0.48。这表明通过改变架构设计（聚合策略）可以有效提升安全性。

### 优点

*   **视角创新：** 将代理型AI的安全问题从“模型对齐”提升到了“系统架构”层面，系统性地分析了多智能体协作带来的结构化风险。这为该领域提供了新的研究思路。
*   **方法论严谨：** 提出了形式化的系统模型和攻击者模型，并引入了一系列新颖且量化的安全指标（妥协概率、CBI、特权漂移、攻击面熵、升级曲率）。反事实的缓解验证（Algorithm 2）设计巧妙，能够有效隔离特定防御措施的因果效应。
*   **实验设计全面：** 从多个维度（智能体数量、攻击者能力、防御策略、模型差异、攻击类型、聚合规则）进行了系统性评估，实验内容丰富且具有说服力。
*   **实用结论：** 结论非常具体且有指导意义，明确指出了“任一代理可执行”是导致风险放大的关键设计缺陷，并提供了“多数共识”这种可落地的防御方案。

### 不足与局限

*   **攻击模型有限：** 攻击模型主要聚焦于“提示注入”和“技能生态（工具）投毒”，对更复杂的、需要多步交互的对抗攻击（如自适应攻击、基于后门的攻击）覆盖不足。
*   **外部有效性有限：** 实验全部基于OpenClaw这一个系统。虽然作者指出结论可能推广到其他存在“推理-执行”耦合的系统，但外部有效性仍有待验证。不同系统的代码、风险点和执行机制差异可能导致不同结果。
*   **未涉及因果机制建模：** 论文发现了特权漂移通过边界不稳定性进行中介的因果关系，但并未对这一因果路径进行严格的数学建模和更深入的分析（例如，使用结构方程模型等）。论文自身也提到这是其局限之一。
*   **对安全和效用的权衡分析不够深入：** 虽然提到了安全措施会降低效用和增加延迟，但并未深入探讨这种权衡在不同场景下的可接受程度，也未提供如何动态调整这种权衡（如自适应安全策略）的方案。
*   **缺乏与其他防御方法的横向对比：** 文中仅评估了OpenClaw内部的防御措施，但没有将OpenClaw的防御效果与其他学术界提出的防御方案（如CaMeL, PromptArmor）在统一的标准下进行比较，这在“工业对比”部分已被作者承认。
