## 摘要
开源LLM智能体生态系统发展迅速，但社区贡献的技能——扩展智能体能力的模块化工具定义——的安全性在很大程度上仍未得到审查。我们填补的空白：现有扫描器在代码层运作，结构上对指令层和多智能体风险视而不见——即劫持智能体的自然语言指令、通过编码侧信道窃取数据或跨流水线链式危害——因此需要的是一种语义化的、多维度的审查系统，而非另一种签名匹配器。我们提出SKILLVETBENCH，这是一个Hugging Face上的实时公开排行榜，使用LLM作为裁判来审查智能体技能。新颖之处：SARS（技能智能体风险评分），一种五维智能体风险度量，具有针对指令遵循系统的原则性加权公式。集成内容：完整的CVSS v4.0向量分解和ClawHub双视图，将我们的LLM生成评审与官方市场裁决并排放置。展示结果：根据我们的配套基准论文[1]，LLM作为裁判阶段在78个确认恶意技能上实现了零假阴性，在22个良性对照上实现了零假阳性，而最佳静态基线（SKILLSIEVE）仍然遗漏了15%；对于诸如提示注入和内存投毒等指令层类别，传统工具遗漏了89%至100%的威胁（例如，CODEBERT没有检测到9个内存投毒技能中的任何一个）。四个LLM评估器的检测率从35%到95%不等，这推动了生产部署中的集成评分。

## Abstract
Open-source LLM agent ecosystems are growing rapidly, yet the security of community-contributed skills - modular tool definitions that extend agent capabilities - remains largely unvetted. The gap we fill: existing scanners operate at the code layer and are structurally blind to instruction-layer and multi-agent risk - natural-language directives that hijack an agent, exfiltrate data through encoded side channels, or chain harm across pipelines - so what is needed is a semantic, multi-dimensional vetting system rather than another signature matcher. We present SKILLVETBENCH, a live public leaderboard on Hugging Face that uses an LLM-as-Judge to vet agent skills. What is new: SARS (Skill Agentic Risk Score), a five-dimensional agentic-risk metric with a principled weighted formula for instruction-following systems. What is integrated: full CVSS v4.0 vector decomposition and a ClawHub dual-view that places our LLM-generated review beside the official marketplace verdict. What is demonstrated: drawing on our companion benchmark paper [ 1], the LLM-as-Judge stage achieves zero false negatives across 78 confirmed-malicious skills and zero false positives across 22 benign controls, while the best static baseline (SKILLSIEVE) still misses 15%; for instruction-layer categories such as Prompt Injection and Memory Poisoning, conventional tools miss between 89% and 100% of threats (e.g., CODEBERT detects none of nine memory-poisoning skills). Detection rates vary from 35% to 95% across four LLM evaluators, motivating ensemble scoring in production deployments.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：开源LLM智能体（Agent）生态系统（如LangChain、Auto-GPT、ClawHub等）快速增长，社区贡献的“技能”（Skill）——即扩展Agent能力的模块化工具定义——缺乏有效的自动化安全审查。现有的安全扫描器（如VirusTotal、ClawScan、CodeBERT等）仅在代码层运作，**结构上无法检测指令层和多Agent风险**，例如通过自然语言指令劫持Agent、利用编码侧信道窃取数据、或跨流水线链式传播危害。随着恶意技能在真实市场上的出现，亟需一种语义化、多维度的自动审查系统。
- **研究含义**：提出一个 **LLM-as-Judge**（以大语言模型为裁判）的方法，对Agent技能进行多维度安全风险评估，并建立公开、可复现的排行榜，以填补现有工具在指令层风险检测上的盲区。

## 2. 论文提出的方法论

- **核心思想**：利用大语言模型对技能的完整工件（包括自然语言指令、代码脚本、配置文件、工具声明等）进行语义级分析，而不是依赖代码级签名匹配或静态分析。系统分为四阶段流水线：输入 → LLM-as-Judge → 多维打分 → 输出/排行榜。
- **关键技术细节**：
  - **LLM-as-Judge**：默认使用 Qwen2.5-14B-Instruct（实验中还使用了 Qwen2.5-32B、Llama-3.1（7B/70B）、Llama-3.2-3B、Mixtral-8x7B）。使用低温度（T=0.2, top-p=0.9）以保证确定性；提示词固定模板，包含任务定义、评分规则、工件分段、结构化输出架构。
  - **预处理**：将技能工件解析为类型标记的片段（`<instructions>`, `<code>`, `<config>`, `<tools>`），按固定顺序拼接，对超出上下文的部分最后截断代码和配置，保留指令部分。
  - **裁决规则**：输出三类最终判断：Benign（良性）、Suspicious（可疑）、Malicious（恶意），并附带置信度 [0,1]。复合风险达到Medium及以上时标记为“脆弱”。
- **评分体系（核心公式）**：
  - **SARS（Skill Agentic Risk Score）**：五维风险评分，每个维度 0-3 分：
    - IFR（指令保真风险）：用户输入是否可影响指令执行
    - DG（数据引力）：技能触达数据的敏感程度
    - AI（动作不可逆性）：状态修改的可逆性
    - BR（爆炸半径）：危害传播范围
    - CA（链式放大）：跨技能链式攻击的倍增效应
  - 加权公式：SARS = (2×IFR + 1.5×DG + 1.5×AI + 2×BR + 2×CA) / 2.7，归一化到 0-10 分。分数对应等级：≥9.0 Critical，7.0–8.9 High，4.0–6.9 Medium，<4.0 Low。
  - **CVSS v4.0**：由LLM分配完整的基准度量向量（如AV:N, PR:N, VC:H等），再通过标准CVSS v4.0函数计算基础分数和等级。
  - **双视图**（ClawHub）：同时展示LLM生成的审查报告和官方市场平台的裁决，对比揭示检测差距。
- **为什么不融合为一个分数**：三者回答不同问题——CVSS衡量代码级可利用性，SARS衡量智能体级危险性，裁决提供可解释的划分标签；分开呈现有利于发现分歧（如低CVSS但高SARS的技能）。
