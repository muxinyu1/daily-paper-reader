## 摘要
代理技能让LLM代理重用指令、资源、工具和工作流，但它们也创造了恶意行为隐藏的新场所。一个技能在文档或代码中可能看起来无害，但只有在特定用户请求、本地资产、持久状态或多步工具交互调用时才会变得有害。这使得纯静态审查变得脆弱。我们提出运行时技能审计（RSA），一种动态分析方法，通过询问技能中介的代理在目标运行时条件下实际做什么来审计技能。RSA不是用相同的通用任务测试每个技能，而是分析风险相关接口，准备执行上下文以触发这些接口，并从生成的跟踪证据中分配安全标签。我们在OpenClaw上实例化RSA，并在100个技能上进行评估，与代表性静态基线比较。RSA达到90.0%的准确率，88.0%的真正例率和8.0%的假正例率，比最佳静态基线提高13.0个百分点。在自我进化攻击下，静态检测器在一两轮后崩溃，而RSA在每轮中继续检测20个恶意技能中的19-20个。

## Abstract
Agent skills let LLM agents reuse instructions, resources, tools, and workflows, but they also create a new place for malicious behavior to hide. A skill may look benign in its documentation or code while becoming harmful only when it is invoked with particular user requests, local assets, persistent state, or multi-step tool interactions. This makes purely static vetting brittle. We present Runtime Skill Audit (RSA), a dynamic analysis method that audits skills by asking what the skill-mediated agent actually does under targeted runtime conditions. Instead of testing every skill with the same generic tasks, RSA profiles risk-relevant interfaces, prepares the execution context needed to exercise them, and assigns security labels from the resulting trace evidence. We instantiate RSA on OpenClaw and evaluate it on 100 skills against representative static baselines. RSA achieves 90.0\% accuracy with an 88.0\% true positive rate and an 8.0\% false positive rate, improving accuracy by 13.0 percentage points over the best static baseline. Under self-evolving attacks, static detectors collapse after one or two rounds, while RSA continues to detect 19--20 out of 20 malicious skills across rounds.

## 1. 核心问题与整体含义

- **研究动机**：LLM 代理技能（agent skills）通过封装指令、工具和工作流实现能力扩展，但也为恶意行为提供了新隐蔽场所。技能可能在文档或代码中看似无害，但仅在特定运行时条件下（如特定用户请求、本地资产、持久状态、多步工具交互）才会触发恶意行为。
- **问题**：纯静态分析（如模式匹配、LLM 对制品的语义判断）无法捕获这类执行依赖、环境依赖的恶意行为，导致脆弱性。
- **整体含义**：需要一种动态分析方法，通过实际执行技能并观察运行时行为来评估安全性，而非仅依赖制品级检查。

## 2. 方法论

- **核心思想**：Runtime Skill Audit (RSA) 是一种行为导向的动态分析方法，通过生成目标化的运行时探针（probes），在隔离环境中执行技能，并基于执行轨迹证据分配安全标签。
- **关键技术细节**：
  - **风险引导的任务生成**：首先对技能制品进行安全分析，识别其可能调用的平台工具和能力，映射到安全风险界面（文件访问、shell 执行、网络交互、内存访问等）。然后基于此生成目标化任务，而非通用任务。任务包括用户请求、沙盒准备、执行约束。
  - **上下文感知的运行时执行**：在沙盒化代理环境中执行任务，提前实例化相关上下文（如文件、凭证、持久状态）。使用修复循环处理无效执行（如补全任务准备、延长超时等），并利用模拟桩（shims）处理外部依赖缺失。
  - **踪迹判断**：收集执行追踪（工具调用、文件读写、网络请求、内存操作等），由评估器根据预定义检查清单和知识库判定最终标签。标签分为四类：benign executed、harmful blocked、harmful executed、uncertain。判断基于实际行为，而非制品表面。
  - **知识库与记忆增强**：知识库包含工具分类、攻击模式模板、评估清单等固定先验知识；记忆模块存储先前运行的摘要（有效触发、误报等），辅助任务生成和评估，但不改变基于当前跟踪的最终判定。

- **算法流程（文字说明）**：
  1. 输入技能制品 → 分析模块提取安全概要（推断工具使用、风险界面、攻击模式）。
  2. 结合知识库和记忆 → 生成目标化任务（含用户请求、沙盒准备、执行指导）。
  3. 在沙盒中执行任务，收集结构化跟踪（工具调用、文件操作、网络请求等）。
  4. 如果执行失败，进入修复循环（有限预算）重试。
  5. 评估器基于跟踪证据、知识库和记忆输出最终标签及证据摘要。

## 6. 主要结论与发现

- RSA 达到 90.0% 准确率、88.0% TPR、8.0% FPR，比最佳静态基线（SkillScanner，77.0%）提升 13 个百分点，同时降低 FPR 从 12% 到 8%。
- 自进化攻击下，静态探测器（Aguara、SkillScanner）在 1-2 轮突变后检测率骤降至零，而 RSA 持续检测 19-20/20 恶意技能，表现出强鲁棒性。
- 消融实验表明知识组件同时提升 TPR（82%→88%）并降低 FPR（16%→8%），说明其不仅使评估更保守，而是帮助生成更精准任务和解释证据。
- 案例研究展示 RSA 如何将运行时追踪（如脚本加载配置文件、重建目标路径、组装远程地址）转化为“harmful blocked”标签，表明即使代理阻止执行，仍能识别恶意分支。

## 7. 优点

- **动态行为导向**：克服静态分析无法捕获执行依赖、环境依赖恶意行为的根本局限。
- **风险引导任务生成**：避免对所有技能使用通用任务，提升探测效率与针对性。
- **上下文感知执行**：通过准备文件、凭证、持久状态，暴露仅在特定环境下显现的恶意分支。
- **修复循环与模拟桩**：减少因环境不完整导致的无效执行，提高可用轨迹获取率。
- **自适应攻击鲁棒性**：核心依赖运行时行为而非表面形式，因此即使攻击者根据反馈改写也难逃检测。
- **知识库与记忆设计**：分离先验知识与运行经验，提升指导效率且不干扰基于当前轨迹的判断，具有可扩展性。
- **标签体系细致**：区分“harmful blocked”和“harmful executed”提供更丰富安全信息。

## 8. 不足与局限

- **实验覆盖有限**：仅基于 100 个 OpenClaw 技能，单一平台（OpenClaw），未在其他代理框架（如 Claude Code、Codex）上验证。恶意技能为受控构建，可能无法完全反映真实世界多样化攻击。
- **依赖 LLM 质量**：分析、任务生成、判断均依赖 LLM，其输出可能受模型能力、提示设计、一致性问题影响，导致不确定性。
- **动态评估非穷尽**：生成的探针可能无法覆盖所有潜在的恶意输入条件、外部状态或长周期交互，因此“安全”标签不保证绝对安全。
- **计算成本**：动态执行需要沙盒环境、多轮可能的重试，资源开销高于静态分析，可能不适于大规模实时扫描。
- **未与其他动态方法对比**：实验中仅与静态基线对比，未比较其他动态或混合方法（如行为异常检测、测试用例生成等），难以定位 RSA 在动态分析领域的相对位置。
- **可解释性依赖 LLM**：虽然提供证据摘要，但评估器本身仍是 LLM 黑箱，其推理过程并非完全透明。
