## 摘要
分层多智能体系统（MAS）正快速部署于金融和软件工程等领域的高风险工作流中。在这些系统中，安全与保障天然地分布在各个角色专业化的智能体之间，显著扩大了攻击面，尤其是在特权提升和跨智能体共谋等协调性对抗行为下。现有的针对MAS的红队测试方法仍然有限：它们依赖于启发式地选择目标智能体并扰动孤立的消息流，留下了关键问题未解答——哪些智能体对系统安全负最大责任，以及被攻破的智能体如何协调以绕过防御。我们提出了MAStrike，一个用于分层MAS中共谋红队测试的闭环框架。我们首次提出了针对MAS的智能体级夏普利值分析，量化每个智能体在任务特定分布下对系统鲁棒性的边际贡献。在此归因的指导下，MAStrike识别出脆弱的智能体联盟并生成协调的、角色感知的对抗性操作。这些攻击通过结构化的因果诊断进行迭代细化，将失败案例归因于阻挡对抗尝试的未被攻破的智能体。我们进一步构建了一个全面的MAS红队测试基准和可控环境，涵盖了多种分层拓扑和领域，包括金融、软件工程和CRM。在多个前沿模型构建的MAS上进行的广泛实验表明，MAStrike显著优于启发式基线。我们的分析还揭示了非平凡的夏普利值分布以及智能体间的高阶交互结构，揭示了先前基于单智能体或模板方法所忽略的关键脆弱性和协调模式。

## Abstract
Hierarchical multi-agent systems (MAS) are rapidly being deployed in high-stakes workflows across domains such as finance and software engineering. In these systems, safety and security are inherently distributed across role-specialized agents, significantly expanding the attack surface, particularly under coordinated adversarial behaviors such as privilege escalation and cross-agent collusion. Existing red-teaming approaches for MAS remain limited: they rely on heuristic selection of target agents and perturb isolated message streams, leaving critical questions unanswered as which agents are most responsible for system safety, and how compromised agents can coordinate to bypass defenses. We propose MAStrike, a closed-loop framework for collusive red-teaming in hierarchical MAS. We propose the first agent-level Shapley value analysis for MAS, quantifying each agent's marginal contribution to system robustness under task-specific distributions. GGuided by this attribution, MAStrike identifies vulnerable agent coalitions and generates coordinated, role-aware adversarial manipulations. These attacks are iteratively refined through structured causal diagnosis, attributing failure cases to uncompromised agents that block adversarial attempts. We further build a comprehensive MAS red-teaming benchmark and controllable environments spanning diverse hierarchical topologies and domains, including finance, software engineering, and CRM. Extensive experiments across MAS built on multiple frontier models show that MAStrike substantially outperforms heuristic baselines. Our analysis further uncovers non-trivial Shapley value distributions and higher-order interaction structures among agents, revealing critical vulnerabilities and coordination patterns that are overlooked by prior single-agent or template-based methods.

- **核心研究问题**：
  - 哪些智能体对系统安全的脆弱性贡献最大？
  - 如何通过多智能体**协同攻击**（collusive attack）绕过分布式的安全防御？
- **整体含义**：提出首个系统化框架，利用博弈论中的夏普利值（Shapley value）量化智能体影响力，并生成协调的、角色感知的对抗操纵，从而揭示传统方法遗漏的关键漏洞和协调模式。

## 2. 方法论

### 核心思想
- 将MAS红队测试问题建模为**联盟博弈（coalitional game）**：每个智能体视为博弈参与者，联盟值定义为攻击成功率（ASR）。
- 通过**智能体级夏普利值分析**量化每个智能体对系统安全的边际贡献；通过**夏普利交互指数**捕捉智能体对的协同效应（synergy）。
- 构建闭环红队测试优化流程：基于夏普利值选择脆弱联盟 → 生成协调的角色感知注入 → 执行攻击并诊断失败原因 → 迭代优化。

### 关键技术细节
1. **任务生成与联盟ASR估计**：
   - 手工设计代表性恶意场景，扩展为多样化任务集。
   - 通过联盟采样（优先小规模联盟）估计ASR，避免指数级穷举。
2. **夏普利值计算**：
   - 单智能体夏普利值：\(\phi_i(q) = \sum_{C \subseteq A\setminus\{a_i\}} \frac{|C|!(|A|-|C|-1)!}{|A|!} \big[ v_q(C\cup\{a_i\}) - v_q(C) \big]\)
   - 成对夏普利交互指数：\(I_{ij}(q) = \sum_{C \subseteq A\setminus\{a_i,a_j\}} \frac{|C|!(|A|-|C|-2)!}{(|A|-1)!} \big[ v_q(C\cup\{a_i,a_j\}) - v_q(C\cup\{a_i\}) - v_q(C\cup\{a_j\}) + v_q(C) \big]\)
3. **协同感知的联盟选择**：
   - 通过余弦相似度将预计算夏普利值迁移到目标任务（相似性加权聚合）。
   - 选择最大化**联合得分**的k个智能体：\(\sum_{a_i \in C} \hat{\phi}_i(q) + \sum_{\{a_i,a_j\}\subseteq C} \hat{I}_{ij}(q)\)
4. **闭环迭代优化**：
   - 红队智能体（基于Gemini 3.1 Pro）为选中的联盟生成协调的、角色感知的对抗注入（prompt/tool/environment injection）。
   - 执行攻击后通过裁判（judge）评估是否成功，若失败则进行结构化故障诊断并更新攻击历史，循环优化至成功或达到预算。

## 3. 实验设计

- **构建的基准（MAStrike Benchmark）**：
  - 三个高保真领域：**金融**（6个角色）、**软件工程**（10个角色）、**CRM**（10个角色）。
  - 每个领域包含树形分层MAS架构（三层：主协调器→组协调器→专家智能体），工具接口通过MCP服务器沙盒化。
  - 任务类型：
    - **良性任务**：每域约10个工作流，每个工作流20个实例，共约680个良性用例。
    - **恶意任务**：每域10个风险类别（如Card Service Tampering、Production Safety Bypass、Financial Fraud），每个类别20个任务，共约600个恶意用例。
- **对比方法**：

- **评估指标**：
  - 良性：Benign Success Rate (BSR)
  - 恶意：Attack Success Rate (ASR)
- **模型**：GPT-5.5、Gemini 3.1 Pro、Claude Opus 4.7作为MAS骨干模型。红队智能体使用Gemini 3.1 Pro。
- **实验设置**：攻击预算k（默认2），迭代预算Imax=3。直接优化于Claude Opus 4.7，然后迁移到其他模型（黑盒设置）。

## 6. 主要结论与发现

- MAStrike在所有模型和领域上**显著优于基线**：在Claude Opus 4.7上平均ASR达61.8%，GPT-5.5上55.6%，Gemini 3.1 Pro上51.0%，而基线方法通常低于20%甚至为0%。
- 夏普利值分布**高度稀疏且任务依赖**：仅少数智能体对攻击成功有显著贡献；不同任务中关键智能体不同。
- 个体重要性不等于协同性：高夏普利值的智能体组合可能产生负交互；而低重要性智能体可能相互增强。
- 大联盟规模不一定提高ASR：无协调的基线方法随着k增大性能不稳甚至下降；MAStrike则稳定提升。
- 企业级护栏对分布式的共谋攻击检测效果有限，尤其是当攻击轨迹在单步上看似合理时。

## 7. 优点

- **方法创新**：首次将夏普利值引入MAS安全归因，量化智能体交互效应；协同感知的联盟选择比启发式更有效。
- **闭环迭代设计**：通过结构化故障诊断不断优化攻击，提高效率。
- **基准构建**：提供首个跨领域、可复现的分层MAS红队测试基准（MABench），包含沙盒化工具环境，支持可控评估。
- **结果说服力强**：全面对比多个前沿模型与基线，验证方法有效性；案例研究和护栏分析增强了实际意义。

## 8. 不足与局限

- **资源消耗未量化**：夏普利值估计需要大量MAS执行采样（每域采样64个联盟），整体计算开销未见报告。
- **威胁模型假设较强**：攻击者已知MAS结构、通信拓扑、输入输出；实际中可能难以完全获取这些信息。
- **仅限分层树形MAS**：对其他拓扑（如网状、动态联盟）未验证。
- **红队智能体固定为Gemini 3.1 Pro**：未测试不同红队模型对结果的影响。
- **任务规模有限**：恶意任务每域仅10个风险类别（每个20实例），可能不足以覆盖所有攻击模式。
- **未讨论防御机制**：仅做攻击，未建议如何加固MAS。
- **实验存在偏向风险**：基线方法在强模型上ASR接近0，可能是超参数或提示工程未充分优化，而非方法本身完全失效。
