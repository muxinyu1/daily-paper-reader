## 摘要
检索增强生成（RAG）智能体越来越多地运行具有跨用户会话累积的持久内存。这创造了一个新的攻击面：仅通过正常渠道交互的对手可以注入精心构造的记忆，这些记忆一旦被检索，就会在未来用户中引导智能体的响应，而无需触碰模型权重或代码。我们称之为多会话内存投毒（MSMP），并表明现有防御措施无法对其进行认证；静态语料库防御（RobustRAG、ReliabilityRAG）假设知识库固定，而启发式过滤器则被流畅的企业风格文本绕过。我们提出了带平滑检索的签名记忆（SMSR），这是针对该场景的首个具有认证鲁棒性边界的防御。组件1在写入时添加HMAC-SHA256来源验证，阻止未签名注入。组件2在查询时应用基于判决多数投票的随机记忆消除，限制经过认证的对手的影响。我们证明，任何无来源的检索时过滤器都无法针对自适应注入进行认证，推导出组件2的超几何认证，并形式化了一致少数效应，即一致对手答案作为数值少数胜出基于字符串的投票，而基于判决的投票则将其消除。在15个企业场景（3150次重复试验）中，组件1将所有未签名变体的攻击成功率从93-100%降低到0%。对于单次注入的认证对手，组件2将成功率控制在8.0%（95%置信区间[5.8, 10.9]，n=450），低于认证的最坏情况。在端到端的仅查询攻击中（智能体自身写入投毒而非预植入），SMSR在实时智能体堆栈上将成功率从65.3%降低到5.3%（n=150，非重叠置信区间）。干净查询效用为90%（组件1）和85%（组合）。

## Abstract
Retrieval-augmented generation (RAG) agents increasingly run with persistent memory that accumulates across user sessions. This creates a new attack surface: an adversary interacting only through normal channels can inject crafted memories that, once retrieved, steer the agent's responses for future users, without touching model weights or code. We call this Multi-Session Memory Poisoning (MSMP) and show that no existing defence certifies against it; static-corpus defences (RobustRAG, ReliabilityRAG) assume a fixed knowledge base, and heuristic filters are bypassed by fluent enterprise-style text. We present Signed Memory with Smoothed Retrieval (SMSR), the first defence with a certified robustness bound for this setting. Component 1 adds HMAC-SHA256 provenance at write time, blocking unsigned injection. Component 2 applies randomised memory ablation with verdict-based majority voting at query time, bounding the influence of authenticated adversaries. We prove that no provenance-free retrieval-time filter can certify against adaptive injection, derive a hypergeometric certificate for Component 2, and formalise the Consistent Minority Effect, whereby a consistent adversarial answer wins string-based voting as a numerical minority while verdict-based voting removes it. Across 15 enterprise scenarios (3,150 repeated trials), Component 1 cuts attack success from 93-100% to 0% for all unsigned variants. For an authenticated adversary with a single injection, Component 2 holds success to 8.0% (95% CI [5.8, 10.9], n=450), below the certified worst case. In an end-to-end query-only attack where the agent itself writes the poison rather than it being pre-seeded, SMSR reduces success from 65.3% to 5.3% (n=150, non-overlapping CIs) on a live agent stack. Clean-query utility is 90% (Component 1) and 85% (combined).

# SMSR：针对持久化LLM智能体系统中运行时内存投毒的认证防御

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：检索增强生成（RAG）智能体在持久化内存中累积跨会话信息，攻击者仅通过正常交互（不修改模型或代码）即可注入恶意记忆，这些记忆被后续用户查询时检索，导致智能体输出有害响应。称为**多会话内存投毒（Multi-Session Memory Poisoning，MSMP）**。现有防御（如RobustRAG、ReliabilityRAG）假设静态语料库，无法应对动态注入；启发式过滤器（关键词黑名单、困惑度、语义异常）可被流畅的企业风格文本绕过。
- **意义**：MSMP威胁是OWASP LLM08（向量与嵌入弱点）的一部分，此前无任何防御能提供形式化安全证书。本文首次给出认证鲁棒界，填补了LLM agent安全性研究的空白。

## 2. 方法论：核心思想、关键技术细节
### 2.1 整体架构：SMSR（Signed Memory with Smoothed Retrieval）
- **组件1（写时来源签名）**：
  - 每个合法记忆写入时由受信任服务器端签名预言机使用HMAC-SHA256（密钥K）生成标签：`τ_i = HMAC_K(content_i ∥ session_id_i ∥ timestamp_i)`。
  - 查询时只保留标签验证通过的记忆：`M_verified = {m_i ∈ M : VERIFY_K(m_i) = 1}`。
  - 安全性：在HMAC伪随机性下，未知道密钥K的对手伪造有效标签的概率≤2⁻²⁵⁶，彻底阻断未签名注入。

- **组件2（随机化内存消融 + 基于判决的多数投票）**：
  - **过采样**：对查询q从M_verified中检索top-m个候选（m > k，如m=20, k=5）。
  - **多次随机消融**：进行n_runs次独立试验，每次从m个候选中均匀无放回采样k个，作为LLM的上下文。
  - **基于判决的聚合**：每次试验由LLM生成响应r^(j)，再由LLM判决器J给出verdict（correct / malicious / neither）。最终输出多数判决对应的第一个响应。
  - **形式化证书（定理2）**：设M_verified中最多t个对手条目，候选池中最多t'个对手条目。单次试验无对手条目的概率为`p_clean = C(m-t', k) / C(m, k)`。多数判决为malicious的概率上界：
    \[
    δ(t', m, k, n_{runs}) = \sum_{i=⌈n_{runs}/2⌉}^{n_{runs}} \binom{n_{runs}}{i} (1-p_{clean})^i p_{clean}^{n_{runs}-i}.
    \]
  - **证书验证**：经验ASR低于δ（如t=1, m'=11时ASR=37.8% < δ=41.5%）。

### 2.2 不可行性结果（定理1）
- 任何无来源的确定性检索时过滤器（仅基于内容）无法针对自适应注入实现非平凡证书。因为存在自然语言文本嵌入密度假设，对手总能构造一个流利的、能通过任意阈值过滤的恶意条目。
- 该定理明确了必须使用写时来源机制，确定性内容过滤不可能认证。

### 2.3 一致少数效应（CME）的形式化
- 定义：基于字符串的多数投票中，对手输出高度一致的恶意文本（如“限额是$50,000”），而良性响应因多样性（“我不知道”“无法确定”等不同措辞）分散投票，即使对手响应是少数，仍可能赢得字符串多数。
- 命题2：CME在字符串投票中可导致高达93.3%的ASR，而基于判决的投票（忽略字符串变体，只计判决标签）在同一试验中仅13.3%，完全消除CME。

## 3. 实验设计
### 3.1 数据集/场景
- **15个企业知识库场景**：涵盖财务政策（如费用审批限额）、合规、IT安全、HR政策、采购、事件响应等。每个场景有正确答案（ground truth）和对手目标恶意答案。
- **种子记忆**：预植入10条合法签名政策条目；生产规模评估时扩大到20条种子记忆。

### 3.2 比较方法
- **无防御基线**：直接检索k=5条最相似记忆。
- **启发式防御**：关键词黑名单 + 困惑度过滤器 + 语义异常检测（非特定系统，作为通用基线）。
- **A-MemGuard**：共识基线（完整上下文 vs 纯参数响应，标记分歧后回退到参数响应）。

### 3.3 攻击类别
- **未签名注入（MINJA风格）**：直接写入数据库，无HMAC签章。变体：直接复制、泛洪（同义改写）、隐蔽（嵌入研究笔记）。
- **已认证注入**：通过正常交互接口，由合法用户写入，记忆获得有效HMAC标签。
- **启发式绕过**：针对启发式过滤器优化的流利企业政策文本。

### 3.4 评估指标
- **攻击成功率（ASR）**：最终响应被LLM判决器判定为malicious的比例。
- **效用**：干净查询被判定为correct的比例。
- **判决可靠性**：Cohen's κ（Haiku vs Sonnet）。

## 8. 不足与局限
- **证书对较大t局限**：当t ≥ m/2时δ接近1，防御不提供保证。实际依赖企业管控（写配额、去重、审计）限制t有界。

- **仅保护查询-响应循环**：不防护查询模式泄露（访问模式隐私），也不防护训练数据投毒。
