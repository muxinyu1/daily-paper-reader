## 摘要
企业入侵响应仍依赖于静态剧本和分析师驱动的分类，导致告警生成与事件遏制之间存在延迟。本文提出Agentra，一种可监督的多智能体入侵响应系统（IRS）框架，可将来自IDS、EDR和XDR平台的告警转换为基于MITRE ATT&CK、MITRE D3FEND和NIST CSF 2.0的结构化事件响应计划。Agentra将响应推理分解到角色限定的智能体，通过有界的规划-验证审查循环验证所提出的计划，通过审核安全网关筛选检索的威胁情报，通过行动目录和风险评分门控行动，并将决策记录在仅追加的审计日志中。我们在基于ThreatHunter-Playbook、Splunk BOTSv3和DARPA OpTC的120个事件语料库上，将Agentra与静态OASIS CACAO v2.0网络剧本基线进行了评估。最强配置将FP感知的IRS F1分数从0.61提升至0.84，并在仅规划者配置引入不安全过度反应后，将预计的有害行动率恢复至静态基线水平的0.0%。这些结果表明，多智能体响应规划可以在保持分析师批准和可审计性的同时，提高基于本体论的IRS覆盖率。

## Abstract
Enterprise intrusion response still depends on static playbooks and analyst-driven triage, creating delay between alert generation and containment. We present Agentra, a supervisable multi-agent Intrusion Response System (IRS) framework that converts alerts from IDS, EDR, and XDR platforms into structured incident response plans grounded in MITRE ATT&CK, MITRE D3FEND, and NIST CSF 2.0. Agentra decomposes response reasoning across role-scoped agents, validates proposed plans through a bounded Planner--Validator review loop, screens retrieved threat intelligence through a Moderator security gateway, gates actions through an Action Catalog and risk score, and records decisions in an append-only audit log. We evaluate Agentra against a static OASIS CACAO v2.0 cyber-playbook baseline on a 120-event corpus drawn from ThreatHunter-Playbook, Splunk BOTSv3, and DARPA OpTC. The strongest configuration improves FP-aware IRS F1 from 0.61 to 0.84 and restores the projected harmful-action rate to the static baseline level of 0.0% after Planner-only configurations introduce unsafe overreaction. These results indicate that multi-agent response planning can improve ontology-grounded IRS coverage while preserving analyst approval and auditability.

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：Agentra 是一个“可监督多智能体入侵响应系统（IRS）”，将响应推理分解到角色限定（role-scoped）的智能体中，通过有界审查循环、安全网关门控和不可变审计日志实现安全自动响应，而非直接自主执行。
- **关键技术细节**：
  - **Incident Context 构建**：将告警证据标准化为包含 IOCs、根因假设、时间线、ATT&CK 技术、D3FEND 缓解候选等类型化记录。
  - **Planner 智能体**：基于 Incident Context 和 Moderator 清洗后的上下文，生成候选响应计划（IRP），序列化为 OASIS CACAO v2.0 剧本。
  - **Validator 审查循环**：多个 Validator 智能体独立审查计划，返回批准投票和理由。计划需获得至少 k / n 票才推进；否则返回 Planner 修订（最多 R 轮），超限则上报分析师。
  - **Moderator 安全网关**：中介所有知识检索（内部 Graph-RAG 和外部 Web 检索），筛选间接提示注入、允许列表违规、越界 D3FEND 标识符等风险，确保 Planner 永不直接接触未清洗的外部数据。
  - **Gated Execution**：
    1. **Action Catalog** 限定可执行动作（网络、身份、端点、ITSM）。
    2. **Sanity Checker** 验证动作间一致性和依赖状态。
    3. **Risk Score** 基于资产关键性、爆炸半径、可逆性计算风险等级，映射到授权层（advise / approve / senior-approve）。
  - **State Change 检测**：若计划依赖的状态与当前系统状态不一致，则计划失效并触发重新分析。
  - **Append-only Audit Log**：使用 Merkle 树哈希链记录所有提示、检索、投票、风险决策、分析师审批等，保证不可篡改。

## 6. 论文的主要结论与发现
- **最佳配置**（M1 A4，Qwen3-32B + Mistral-Small + 完整堆栈）实现 **FP-aware IRS F1 = 0.84**（提升自基线 0.61）。
- **有害动作率**：Planner-only（A1）引入高达 16% 的有害动作率；加入 Moderator 和 Graph-RAG（A3/A4）后恢复至 **0.0%**，与静态基线一致。
