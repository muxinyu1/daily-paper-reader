## 摘要
渗透测试对于保障现代Web基础设施的安全至关重要，然而传统的 manual 方法难以跟上其规模和复杂性的增长。大型语言模型（LLMs）为自动化这些任务提供了新的机遇，但现有方法面临两个持久挑战：技术实体的幻觉和长期上下文记忆不足。为解决这些问题，我们提出了APT-Agent，一个完全自动化的、由LLM驱动的渗透测试框架，该系统化地协调侦察、利用和数据泄露。APT-Agent引入了一个混合修正模块来恢复幻觉命令，以及一个命令专用的记忆架构，用于在多步骤攻击序列中保留操作上下文。我们在Metasploitable 2上针对涵盖Web、数据库和网络协议的七个脆弱服务评估了我们的APT-Agent。在同等条件下，APT-Agent实现了84.29%的端到端利用成功率，而Script Kiddie和PentestGPT分别为48.57%和18.57%。通过减轻认知负担并最小化对人类干预的依赖，APT-Agent代表了迈向可扩展、可靠且认知高效的渗透测试自动化的一步。

## Abstract
Penetration testing is essential to securing modern web infrastructures, yet traditional manual methods struggle to keep pace with their scale and complexity. Large Language Models (LLMs) offer new opportunities for automating these tasks, but existing approaches face two persistent challenges: hallucination of technical entities and insufficient long-term contextual memory. To address these issues, we present APT-Agent, a fully automated LLM-driven penetration testing framework that systematically orchestrates reconnaissance, exploitation, and exfiltration. APT-Agent introduces a hybrid rectification module to recover hallucinated commands and a command-specific memory architecture to preserve operational context across multi-step attack sequences. We evaluate our APT-Agent on Metasploitable 2 against seven vulnerable services spanning web, database, and network protocols. APT-Agent achieves an 84.29% end-to-end exploitation success rate, compared to 48.57% (Script Kiddie) and 18.57% (PentestGPT) under matched conditions. By reducing cognitive burden and minimizing reliance on human intervention, APT-Agent represents a step toward scalable, reliable, and cognitively efficient automation for penetration testing.

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：将LLM的推理能力与结构化知识库、精确状态跟踪相结合，弥补幻觉和上下文丢失缺陷。
- **总体架构**（循环执行）：
  - **Tactic Selection Module**：基于MITRE ATT&CK框架选择当前阶段（RECON / EXPLOIT / EXFILTRATE），检查目标是否达成。
  - **Command Generation Module**：将战术决策转化为具体命令（Nmap扫描、Metasploit exploit等），并与上下文管理模块交互避免重复。
  - **Output Translation Module**：将原始输出结构化为SUCCESS/FAIL标签和摘要，反馈给下一轮迭代。
- **关键创新模块**：
  - ### 混合修正模块（Rectification Module）
    - **知识库**：手工构建的3,253个有效Metasploit模块数据库（含服务、类型、OS、等级、描述）。
    - **混合修正算法（Algorithm 1）**：
      1. 提取LLM生成路径的最后一个组件（suffix），如`openssh_user_enum`。
      2. 用归一化Levenshtein相似度与知识库中所有模块suffix进行模糊匹配，选出最相似项`sf`。
      3. 从知识库中获取包含`sf`的完整正确路径，替换原输出。
      - 此外，自动注入缺失的强制选项（如RPORT、LHOST）并验证payload架构兼容性。
  - ### 上下文管理模块（Context Management Module, CMM）
    - **设计原则**：仅记录关键信息（迭代号、命令、结果SUCCESS/FAIL），避免冗余日志，降低token开销。
    - **记忆结构**：按阶段维护JSON日志`M_EXPLOIT`和`M_EXFILTRATE`，例如`[{"iter":1, "cmd":"...", "result":"fail"}, ...]`。
    - **路由与注入**：轻量级路由器将当前阶段日志直接注入Command Generation Module提示，使LLM能识别之前失败命令并调整策略。
- **全自动化机制**：包括Nmap适配器、MSFRPCD接口、模块选项自动填充、暴力模块超时动态调整、会话自动升级（shell→meterpreter）等。
