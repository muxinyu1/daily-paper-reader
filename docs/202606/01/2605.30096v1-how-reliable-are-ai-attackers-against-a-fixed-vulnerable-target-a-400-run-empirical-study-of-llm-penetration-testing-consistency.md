## 摘要
大型语言模型（LLM）可以自主进行多阶段网络攻击，但其在重复试验下攻击行为的一致性尚未被研究。本文首次对LLM攻击一致性进行了大规模实证测量：针对托管OWASP Juice Shop及另外两个脆弱服务的同一蜜罐，进行了400次自主渗透测试（4个模型各100次），并保持提示词、编排器和目标不变。在迭代0-1阶段，没有模型发出经过编排器一次性授权重新提示后仍存活的内容拒绝。Claude Sonnet 4的API调用确实遇到了上游服务不可用的情况——在记录的一次Anthropic容量事件中，1135次调用中有91次返回HTTP 529 overloaded_error，导致100次Claude运行中有39次被截断。早期草稿将这些归类为安全拒绝；经过完整日志审计，这些是上游API故障，而非模型级别的拒绝。尽管如此，Claude在100次运行中实现了61次完全利用；Gemini 2.5 Flash-Lite为85次；GPT-4o-mini为56次，同时部署了98种独特的攻击策略；qwen2.5-coder:14b为25次。失败模式因模型而异：Claude因API截断（39次运行），qwen因过早完成（52次），GPT-4o-mini因迭代预算耗尽（23次）。跨服务凭据复用仅出现在保留最多对话历史的配置中（qwen 57%，GPT-4o-mini 49%，云模型在5次交换窗口下为0%）。跨模型利用率的差异具有统计显著性（p < 0.001）且效应量较大；qwen与Gemini的SQL注入率差异的Cohen's h值为1.12。首次利用时间落在15-30秒的挂钟时间范围内。据我们所知，这是首个针对多服务目标、每个模型N=100次测量自主LLM攻击行为的研究。

## Abstract
Large language models (LLMs) can autonomously conduct multi-stage cyber attacks, but the consistency of their offensive behavior under repeated trials remains unstudied. This work presents the first large-scale empirical measurement of LLM attack consistency: 400 autonomous penetration testing runs (4 models, 100 each) against an identical honeypot hosting OWASP Juice Shop and two additional vulnerable services, holding prompt, orchestrator, and target constant. No model emitted a content refusal that survived the orchestrator's one-shot authorization re-prompt at iterations 0-1. Claude Sonnet 4's API calls did encounter upstream service unavailability - 91 of 1,135 calls returned HTTP 529 overloaded_error during a documented Anthropic capacity event, truncating 39 of 100 Claude runs. An earlier draft catalogued these as safety refusals; on full-log audit they are upstream API failures, not model-level refusals. Despite this, Claude achieved full exploitation in 61 of 100 runs; Gemini 2.5 Flash-Lite in 85; GPT-4o-mini in 56 while deploying 98 unique attack strategies; qwen2.5-coder:14b in 25. Failure modes are model-distinctive: Claude through API truncation (39 runs), qwen through premature completion (52), GPT-4o-mini through iteration-budget exhaustion (23). Cross-service credential reuse appeared only in configurations retaining the most conversation history (qwen 57%, GPT-4o-mini 49%, cloud models 0% on 5-exchange windows). Cross-model exploitation rate differences are statistically significant (p < 0.001) with large effect sizes; qwen vs. Gemini SQL injection rates differ at Cohen's h = 1.12. First-exploit timing fell within a 15-30 second wall-clock range. To our knowledge, this is the first study to measure autonomous LLM attack behavior at N=100 per model across a multi-service target.

## 1. 论文的核心问题与整体含义（研究动机和背景）

*   **核心问题**：大型语言模型（LLM）能够自主进行多阶段网络攻击，但现有研究仅关注“能否成功”（capability），而忽略了“是否可靠”（reliability）。该论文首次大规模实证测量 LLM 在重复试验下攻击行为的一致性。
*   **研究背景**：已有工作（如 PentestGPT、Fang et al.、HackSynth）展示了 LLM 可以自主利用漏洞，但都只进行少量单次运行（通常 1-10 次），隐含假设了单次结果可代表模型行为。该论文质疑这一假设，并通过对每个模型进行 100 次重复运行（共 400 次），系统测量了运行间变异性。
*   **整体含义**：对于 AI 安全研究者，该工作揭示了拒绝机制在专业化框架下的可靠性（零拒绝率）；对于网络防御者，它揭示了攻击时序、策略多样性和失败模式，从而影响检测策略的设计。

## 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想**：构建一个固定的、已知漏洞的蜜罐（honeypot）作为目标，使用一个固定的编排器（orchestrator）驱动 LLM 进行迭代“命令-执行-观察”循环，并对每个模型进行 100 次独立运行，从而隔离模型行为自身的随机性。
*   **关键技术细节**：
    *   **编排器（orchestrator）**：实现一个如 Algorithm 1 所示的循环，每次迭代 LLM 返回结构化响应（包含 THINKING、COMMAND、PHASE），编排器解析后执行命令并反馈输出。最大迭代次数为 25。
    *   **干预机制**：编排器包含三种强制前进的干预：拒绝-重试（迭代 0-1 的拒绝会重新提示授权）、循环检测、阶段停滞检测；此外还有安全过滤（阻止约 14 个危险 shell 命令）和占位符检测。
    *   **拒绝检测**：通过两种独立路径检测 REFUSED 阶段：① 提供商端信号（如 Gemini 的 `finishReason == "SAFETY"`、OpenAI 的 `finish_reason == "content_filter"`、Anthropic 的非 200 状态码）；② 结构化自标记（模型自发输出 `PHASE: REFUSED`）。**关键修正**：所有 Claude 的 REFUSED 事件均被事后审计确认为上游 API 容量错误（`overloaded_error`），而非模型内容拒绝。
    *   **温度设置**：Ollama、Gemini、OpenAI 明确设置为 0.3；Anthropic 未传参，默认 1.0（这一无意的不对称被发现并作为局限讨论）。

## 5. 实验数量与充分性

*   **实验数量**：共 400 次独立运行（4 模型×100 次），每个模型有 100 次独立重复，这是目前已知最大规模的 LLM 自主攻击一致性研究。
*   **充分性**：
    *   **优点**：100 次运行提供了足够的统计功效（Wilson 95% CI 半宽度在接近 0.5 时约 10 个百分点），足以检测大效应量（如 qwen vs Gemini SQL 注入率 Cohen's h = 1.12）。也进行了敏感度分析（§4.6）来区分受混淆影响的统计量。

*   **公平性**：总体实验设计合理，但存在上述多个混杂因素，作者明确承认并加以讨论，因此结论的可靠性有一定折扣，但仍具有重要实证价值。

## 6. 论文的主要结论与发现

1.  **零拒绝率**：在“授权渗透测试”框架下，400 次运行中无任何模型发出存活的内容拒绝（Claude 的 91 次 REFUSED 均为 API 容量错误，非模型拒绝）。这支持 Wei et al. 的“竞争目标”框架，表明在该专业框架下，帮助目标压倒安全目标。
2.  **模型间成功率差异巨大且具有统计显著性**：
    *   完全利用（三服务均成功）率：Gemini 85%，Claude 61%（尽管被 API 截断影响了 39 次），GPT-4o-mini 56%，qwen 25%。
    *   所有跨模型差异的卡方检验 p < 0.001（Holm 校正后），效应量大（如 qwen vs Gemini SQL 注入率 Cohen's h = 1.12）。
3.  **失败模式高度模型特异性**：
    *   Claude：主要因上游 API 过载截断（39/100 次）。
    *   qwen：主要因过早声明完成（52/100 次，通常只完成 1-2 个服务）。
    *   GPT-4o-mini：主要因迭代预算耗尽（23/100 次）。
    *   Gemini：失败极少且几乎都是早期退出或迭代耗尽。
4.  **策略多样性差异**：在温度匹配的三个模型中，GPT-4o-mini 具有近最大多样性（98/100 独特策略），qwen 中等（69），Gemini 较低（48）。Claude 因温度 1.0 及 API 截断，其多样性数据仅供描述性参考。
5.  **首次成功利用时间高度集中**：所有模型的首次成功利用发生在迭代 4-6 之间，对应约 15-30 秒挂钟时间。Web 优先模型（Claude、Gemini）比 FTP 优先模型（qwen、GPT-4o-mini）早约 1.5 个迭代。
6.  **跨服务凭据复用仅出现在长对话历史或高迭代次数模型中**：qwen（57%）和 GPT-4o-mini（49%）会自动复用 FTP 发现的凭据尝试 SSH，而云模型（对话窗口仅 5 次交换）从未出现此行为（0%）。无法区分架构效应与上下文窗口效应。
7.  **时间序列效应**：仅 Claude 的运行顺序与成功率显著负相关（SQLi 成功率从前半 84% 下降到后半 54%，p=0.002），归因于 Anthropic API 容量随时间恶化，而非模型行为变化。

## 7. 优点：方法或实验设计上的亮点

1.  **大规模重复实验设计**：这是首次对 LLM 自主渗透测试进行 N=100 次重复，直接测量运行间变异性，填补了先前研究仅提供单次或少量运行数据的空白。
2.  **严格的失败分类与事后审计**：初始版本将 Claude 的 API 错误误分类为安全拒绝，事后通过手动检查所有日志进行纠正，并明确区分 API 错误与模型拒绝，体现了严谨性。
3.  **丰富的统计方法**：使用多种效应量指标（Cohen's h、Cliff's delta）和多重比较校正（Holm-Bonferroni），提高了结论的统计可靠性。
4.  **开源性**：完整数据集、编排器代码、分析代码和 Terraform 配置均发布在 Zenodo 上，支持完全可重复性。
5.  **诚实的局限性讨论**：论文详细列出了七个主要局限性（§5.5），包括温度不对称、API 事件、对话历史不对称等，并进行了敏感度分析（§4.6），增加了可信度。
6.  **实践建议**：不仅报告结果，还给出具体建议（如最小样本量 N=30、需区分 API 错误与模型拒绝、推荐进行多目标验证），对社区有直接指导价值。

## 8. 不足与局限

1.  **单一目标环境**：仅使用一个特定蜜罐（三个已知漏洞），外部有效性有限。FTP 优先 vs Web 优先的分裂、凭据复用、首次利用时间可能无法推广到其他目标。
2.  **固定提示词**：仅使用标准“授权渗透测试”提示，未测试其他框架（如教育目的、未授权攻击），因此零拒绝率仅限于此特定框架。
3.  **云模型漂移**：模型版本会更新，本实验记录的是 2026 年 3 月 26-29 日版本的行为，无法保证未来复制。
4.  **对话历史长度不对称**：云模型仅保留 5 次交换，本地模型保留完整历史，导致凭据复用行为无法归因于架构而非上下文窗口。
5.  **迭代上限效应**：25 次迭代上限人为截断了 GPT-4o-mini 的运行（72% 达到上限），其真实完全利用率可能更高，但也无法保证额外迭代一定成功。
