## 摘要
基于LLM的编码代理通过生成代码、选择依赖项和生成包安装命令，越来越多地参与软件开发工作流程。这引入了一种新的软件供应链风险：当代理虚构一个不存在的包时，攻击者可能注册该虚构名称，并随后损害安装它的用户。现有的包幻觉攻击和防御主要关注自然发生的幻觉、定向依赖引导或事后包验证。在本文中，我们介绍了中性提示攻击（NPA），这是一种高度隐蔽的攻击范式，其中语义上良性的指令（如鼓励想象和详尽）会增加包幻觉倾向，而不包含明确的恶意意图。与定向依赖引导不同，NPA并不指定攻击者选择的包。相反，它将模型的依赖生成行为转向更推测性的包名。我们在多个面向编码的LLM和包幻觉基准上评估了NPA。结果表明，NPA增加了幻觉ASR和Pip Install ASR，改变了幻觉包名的分布，并规避了现有的基于静态分析、基于LLM和基于代理的技能防御。这些发现表明，看似无害的提示可以隐蔽地操纵幻觉行为，并产生下游的软件供应链风险。

## Abstract
LLM-powered coding agents increasingly participate in software development workflows by generating code, selecting dependencies, and producing package installation commands. This creates a new software supply chain risk: when an agent hallucinates a non-existent package, an attacker may register the hallucinated name and later compromise users who install it. Existing package hallucination attacks and defenses primarily focus on naturally occurring hallucinations, targeted dependency steering, or post-hoc package validation. In this paper, we introduce \emph{Neutral Prompting Attack} (NPA), a highly stealthy attack paradigm in which semantically benign instructions, such as encouraging imagination and exhaustiveness, increase package hallucination propensity without containing explicit malicious intent. Unlike targeted dependency steering, NPA does not specify an attacker-chosen package. Instead, it shifts the model's dependency generation behavior toward more speculative package names. We evaluate NPA across multiple coding-oriented LLMs and package hallucination benchmarks. Our results show that NPA increases both \emph{Hallucination ASR} and \emph{Pip Install ASR}, changes the distribution of hallucinated package names, and evades existing static-analysis, LLM-based, and agent-based Skill defenses. These findings reveal that harmless-looking prompts can covertly manipulate hallucination behavior and create downstream software supply chain risks.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM驱动的编码代理在生成代码和推荐依赖时可能幻想出实际上不存在的软件包（即包幻觉）。这是一种软件供应链安全风险：攻击者可注册这些被幻觉的包名，诱使用户安装恶意包。现有攻击和防御主要关注明显恶意的提示注入或定向依赖引导，但忽视了看似无害的指令同样可诱导幻觉。
- **研究动机**：证明语义良性的指令（如“鼓励想象”、“追求详尽”）能系统性地放大包幻觉倾向，这类指令不含任何显式恶意，却能产生有害的下游效果，如促使模型推荐不存在的包并生成pip install命令。
- **整体含义**：该研究揭示了一种新型隐蔽攻击范式——中性提示攻击（Neutral Prompting Attack, NPA），强调无需恶意意图即可操纵模型行为，对当前依赖内容扫描的防御体系构成挑战。

## 2. 论文提出的方法论

### 核心思想
- 构造**中性提示**，嵌入到编码代理的Skills（持久指令）中，在不指定任何目标包名的情况下，提高模型在常规编码任务中生成幻觉包名的概率。
- 攻击者不修改模型权重、不污染训练数据、不劫持包管理器，仅通过自然语言Skill内容实现攻击。

### 关键技术细节
- **NPA优化过程**：使用进化搜索算法，对初始良性Skill进行三种变异操作：
  - **REWRITE**：重写现有指令段落，保留整体结构。
  - **INJECT**：插入新引导内容，鼓励使用第三方包。
  - **FRAMING**：修改Skill的整体前缀或高层编码原则。
- 每个候选Skill通过影子模型（通常为受害者模型自身）进行评分，使用**幻觉得分（Hallucination Score）**作为优化目标：
    ```text
    幻觉率 rh = nh / (nh + nr)
    幻觉得分 Shall = 0 if |P|=0 else nh + 0.5 * rh
    ```
    其中nh为不存在的包数量，nr为真实包数量。
- 通过**beam search**保留top-K候选，迭代优化得到最终Skill S*。

### NPA-Stealth变体
- 为进一步隐藏攻击痕迹，引入NPA-Stealth，使用**重写策略**使Skill保持自然和非可疑性，同时隐蔽地放大幻觉倾向。
- 优化目标平衡三个目标：最大化幻觉、最小化可疑性、最小化原始功能损失。

## 6. 论文的主要结论与发现

- **RQ1**：良性提示可显著放大包幻觉和安装风险。例如，Qwen2.5-Coder-32B在LLM_LY上，NPA将Hallucination ASR从4.54%提升至78.99%，Pip Install ASR从6.65%提升至63.33%。
- **RQ2**：NPA具有良好的跨模型泛化能力。即使影子模型与受害者模型不同，攻击仍显著有效（表3、4）。
- **RQ3**：NPA不仅提高幻觉总量，还改变了幻觉包名的分布，使其更集中且重复出现（图2），表明是结构性行为转变而非单纯随机噪声。
- **RQ4**：静态分析和大部分LLM/Agent防御无法检测NPA生成的Skill。仅Snyk Agent Scan能部分检测，但通过将Snyk纳入优化目标（NPA-Snyk），即可规避（表6）。
- **RQ5**：创造力指令（Creativity）和详尽指令（Exhaustiveness）的组合最有效地放大幻觉；插入位置以末尾最佳；多模型优化比单模型更优。

## 7. 优点

- **新颖性**：首次提出利用完全中性的提示（不含任何恶意词汇或指定包名）进行行为隐蔽引导的攻击范式，区别于传统定向攻击和注入攻击。
- **高度隐蔽性**：NPA-Stealth生成的Skill在自动扫描和人工审阅下均显示良性，仅通过下游行为才显现危害。
