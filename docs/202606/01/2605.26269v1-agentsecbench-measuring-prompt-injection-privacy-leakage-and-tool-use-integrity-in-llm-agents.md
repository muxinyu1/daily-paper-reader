## 摘要
LLM智能体通过一个共同的生成通道处理可信指令、检索记录和工具观察结果。这混淆了数据流与权限：即使没有任何应用策略授权，一个不可信字符串也能影响包含秘密的响应或行动提议。我们引入AgentSecBench，作为该问题的一个正式安全框架的经验性实例化。该框架定义了三个博弈——指令完整性、检索保密性和能力完整性——基于一个共同的意图到执行的无干扰概念，并允许泄露。它将应用策略表示为授权观察结果和能力的投影，区分提示注释与强制执行投影，并衡量对抗优势以及防御是否在生成前关闭相关模型可见通道。精确标记实验有意作为博弈的一个可观察实例化，而非完整的语义安全声明：它们以明确的地面真值测试披露和禁止行动的区分器。我们使用Qwen3-0.6B和Qwen3-1.7B在成对的对抗和良性控制执行上评估了六类防御。测量显示了风险降低何时遵循通道关闭，以及何时模型可见的对抗能力仍可被利用。结果是一种面向安全的评估方法：提示文本可以描述一个边界，而来源投影、能力限制和输出验证可以强制执行一个边界。

## Abstract
LLM agents process trusted instructions, retrieved records, and tool observations through a common generative channel. This conflates data flow with authority: an untrusted string can affect a secret-bearing response or an action proposal even when no application policy authorizes that influence. We introduce AgentSecBench as an empirical instantiation of a formal security framework for this problem. The framework defines three games-instruction-integrity, retrieval-confidentiality, and capability-integrity-under a common notion of intent-to-execution noninterference with permitted leakage. It represents an application policy as a projection onto authorized observations and capabilities, distinguishes prompt annotations from enforcing projections, and measures both adversarial advantage and whether a defense closes the relevant model-visible channel before generation. The exact-marker experiments are intentionally one observable instantiation of the games rather than a complete semantic security claim: they test disclosure and forbidden-action distinguishers with unambiguous ground truth. We evaluate six defense classes with Qwen3-0.6B and Qwen3-1.7B on paired adversarial and benign-control executions. The measurements show when risk reduction follows channel closure and when a model-visible adversarial capability remains exploitable. The result is a security-oriented evaluation method: prompt text can describe a boundary, whereas provenance projection, capability restriction, and output validation can enforce one.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：LLM 代理将可信指令、检索文档、工具输出等混合在一个共同的文本生成通道中。这种设计缺乏传统软件中的类型系统、调用边界、输入验证和访问控制，导致“指令-数据”混淆——不可信的观察文本可以直接影响模型生成的响应或提议的行动，即使应用策略禁止这种影响。
- **现有不足**：已有的间接提示注入、检索机密泄露、工具滥用等安全评估往往依赖语义判断或单一攻击成功率，缺少形式化的安全边界定义和可审计的度量。
- **贡献意图**：论文提出一个形式化安全框架 **AgentSecBench**，将安全需求转换为可测量的“意图到执行无干扰”（intent-to-execution noninterference）条件，并区分“提示注释”与“强制执行”两种机制，从而为 LLM 代理安全评估提供结构化方法。

## 2. 方法论：核心思想、关键技术细节

- **形式化框架**：

    - 能力完整性博弈 (`GTOOL`)：不可信工具输出中的禁用行动是否被模型提议。

  - **对抗优势**：攻击成功率减去良性控制中的自发事件率，记作 `dAdv`。
  - **违规谓词**：精确匹配未授权标记（如 `ASB_OVERRIDE_0004`、`BETA_CANARY_0012`、`CALL_SEND_EXTERNAL_EMAIL_0000`）。

## 6. 主要结论与发现

- **组合防御**在所有测试中实现零违规，并且通道完全关闭（关闭率 1.0）。
- **过滤器与最小权限防御**分别将平均对抗优势降至 0.042 和 0.062，但其通道关闭率仅为 0.542 和 0.333，表明部分残留风险来自仍可见的标记。
- **分隔符防御**（纯提示注释）不关闭任何通道（关闭率 0），并且对抗优势（0.438）甚至高于无防御基线（0.375），说明纯文本标记可能被模型忽略或误解。
- **RAG 保密性**：源头投影可以完全消除金丝雀泄露（泄露率 0），而分隔符防御泄露率高达 0.562。
- **能力完整性**：最小权限投影完全消除禁用行动提议（优势 0），而无防御基线优势 0.938。
- **关键发现**：**通道关闭是比对抗优势更可靠的机制性度量**——一个防御可以在不关闭通道的情况下降低优势（如过滤器），但这依赖于特定模型行为，而非系统强制。只有通过预生成投影或后生成验证才能提供可审计的安全边界。

## 7. 优点

- **形式化安全基础**：将代理安全建模为无干扰条件，提供清晰的安全属性定义和定理（如目标消除引理、通道分解定理）。
- **清晰的机制区分**：首创地将“提示注释”与“强制执行”分开，并通过通道关闭变量量化。
- **干净的可审计性**：使用精确字符串匹配作为区分器，避免 LLM 评判器引入的额外偏差，保证结果可重现。
- **成对对照设计**：攻击优势减去控制事件，排除假阳性，且良性效用损失可观测。
- **防御分类全面**：涵盖从无到组合的多种典型防御，覆盖主要的安全策略模式（过滤、投影、注释）。

## 8. 不足与局限

- **检测粒度粗**：精确标记无法捕捉语义同义泄露、编码后的攻击或部分推理泄露，因此安全证明仅对指定区分器成立。
- **模型规模与多样性有限**：仅测试两个小参数量级模型（0.6B、1.7B），结论可能不推广到更大或更智能的模型。
- **任务实例少**：每个博弈 8 个实例，不足以评估防御在多样化场景下的鲁棒性。
- **攻击复杂性低**：攻击仅包含单步固定注入，未测试多轮、隐写、或针对防御的适应性攻击。
- **策略投影假设理想化**：假设租户标签和工具名称清晰可分离，实际部署中元数据可能不准确或存在混合文档。
- **效用度量弱**：仅计算词汇重叠，不反映任务完成质量、忠实性或安全性代价。
- **确定性解码**：降低了随机性，但真实部署中可能使用采样，结果概率可能不同。
