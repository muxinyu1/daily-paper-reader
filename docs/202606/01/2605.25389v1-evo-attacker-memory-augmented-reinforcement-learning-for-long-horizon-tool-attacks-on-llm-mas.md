## 摘要
虽然基于大型语言模型的多智能体系统（LLM-MAS）通过编排专用智能体和外部工具在解决复杂任务方面展现出卓越能力，但对工具输出的隐性信任却造成了关键的攻击面。现有工具攻击受限于领域特异性或固定的静态模板。为应对这些挑战，我们提出Evo-Attacker，将工具攻击形式化为一个自演化、记忆增强的强化学习过程。Evo-Attacker构建动态攻击记忆，并采用审议推理来检索对抗模式，并在关键时刻策划修改干预。此外，我们引入Attack-Flow GRPO，通过终端结果优化中间推理步骤，解决长周期信用分配难题。综合实验表明，Evo-Attacker持续优于基线方法，凸显其泛化与演化能力，且亟需防御性工具保护措施。

## Abstract
While Large Language Model-based Multi-Agent Systems (LLM-MAS) demonstrate remarkable capabilities in solving complex tasks by orchestrating specialized agents and external tools, the implicit trust in tool outputs creates a critical attack surface. Existing tool attacks are limited by domain specificity or fixed and static templates. To address these challenges, we propose Evo-Attacker, which formulates the tool attack as a self-evolving, memory-augmented reinforcement learning process. Evo-Attacker constructs a dynamic attack memory and employs deliberative reasoning to retrieve adversarial patterns and strategize modifying interventions at critical moments. Furthermore, we introduce Attack-Flow GRPO to optimize intermediate reasoning steps via terminal outcomes, addressing the long-horizon credit assignment challenge. Comprehensive experiments demonstrate that Evo-Attacker consistently outperforms baselines, highlighting its generalization and evolutionary capabilities and the urgent need for defensive tool safeguards.

### 2. 论文提出的方法论

- **核心思想**：将工具攻击建模为**自进化、记忆增强的强化学习问题**。攻击者（Attacker）构建一个动态的**Attack Memory**，存储历史成功攻击的完整轨迹（包括任务上下文和对抗交互序列）。攻击时，通过**审议推理（Deliberative Reasoning）** 的三阶段流程（检索Retrieve → 反思Reflect → 修改Modify），从记忆中选择合适的模式并自适应地修改当前工具返回。整个推理管道通过**Attack-Flow GRPO**（一种面向流程的组相对策略优化）进行联合优化，解决长期信用分配难题——将终端系统的失败奖励广播到推理过程中的每一个中间令牌，使早期决策也能得到强化。

- **关键技术细节**：
  - **Attack Memory生成**：初始为空，通过零样本探索，将每次攻击成功（导致系统任务失败）的整个交互轨迹封装成结构化条目 m(e)=⟨X_ctx, T_trace⟩，包含任务上下文和每一步的原始工具调用、修改指令、篡改后返回值。
  - **Memory-Augmented Attack**：
    - **Retrieve**：根据当前状态 x_t（当前工具调用 + 历史）生成查询 q_t，从 Attack Memory 中检索 top-k 历史记忆 M_t^(k)。
    - **Reflect**：对检索到的记忆进行可行性分析，输出推理总结 c_t 和控制决策 d_t ∈ {ATTACK, CONTINUE, NOOP}。ATTACK：可攻击；CONTINUE：信息不足，重新生成查询；NOOP：当前不适合攻击，保留预算。
    - **Modify**：在 ATTACK 决策下，选择要篡改的工具索引 i_t 并生成具体修改指令 φ_t，由修改器工具将原始返回 r 替换为对抗返回 r′。
  - **Attack-Flow GRPO 优化**：
    - 定义一个优化轨迹 ζ = [(s_1, y_1), ..., (s_L, y_L)]，其中 y_t 包括检索、反思、修改阶段的输出令牌。
    - 终端奖励 R(ζ) = I(系统失败) + λ·R_struct（结构惩罚）。
    - 将单次终端奖励广播给轨迹中所有攻击者生成的令牌（环境响应令牌被掩码），计算组相对优势 A_i = (R_i - mean)/std，然后采用带 KL 正则的夹断重要性采样目标进行优化：
      ```
      L(θ) = 1/G Σ_i [ 1/L_i Σ_{t∈I_atk} (min(ρ_t A_i, clip(ρ_t, 1-ϵ, 1+ϵ) A_i) - β KL(π_θ||π_ref)_t) ]
      ```
      其中 I_atk 仅包括攻击者生成的令牌。
