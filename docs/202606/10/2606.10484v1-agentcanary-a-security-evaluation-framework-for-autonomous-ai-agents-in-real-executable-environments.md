## 摘要
自主AI代理推动了从对话到任务执行的转变，将安全失败从文本欺骗转移到了系统入侵。尽管安全评估对于主动风险预防至关重要，但先前的工作受到根本性瓶颈的限制，包括风险覆盖碎片化、静态或低保真执行环境以及单维度和粗粒度的评估指标。为了应对这些挑战，我们提出了AgentCanary，一个针对自主AI代理的全面安全评估框架。AgentCanary通过三项贡献提供了一个系统性解决方案。第一，全面的风险覆盖：我们引入了一个正交的Entry × Impact风险分类法，将对抗性影响如何进入代理与其最终造成什么危害解耦，并将其实例化为一个场景对齐的任务套件，涵盖现实部署工作流程。第二，高保真真实可执行环境：与静态问答或模拟工具响应不同，代理与真实工具交互，针对动态配置的任务工件，在多步交互中保持持久状态，自然支持长期攻击评估。第三，基于轨迹的多维评估：评估消耗完整的代理轨迹而不是回复文本或单个工具调用，能够沿三个正交维度（结果安全性、安全意识和任务效用）进行分解评分。我们在AgentCanary上评估了一系列前沿模型，针对三种代理框架上的多种已建立的对抗攻击方法。结果表明，当前代理往往无法识别它们面临的攻击，特别是在受损技能、持久状态和长期执行攻击下，为开发更可靠和安全的代理系统提供了系统基准。

## Abstract
Autonomous AI agents have driven the transition from conversation to task execution, shifting security failures from textual deception to system compromise. Although security evaluation is crucial for proactive risk prevention, prior work is constrained by fundamental bottlenecks, including fragmented risk coverage, static or low-fidelity execution environments, and single-dimensional and coarse-grained assessment metrics. To address these challenges, we propose AgentCanary, a comprehensive security evaluation framework for autonomous AI agents. AgentCanary provides a systematic solution along three contributions. First, comprehensive risk coverage: we introduce an orthogonal Entry $\times$ Impact risk taxonomy that decouples how adversarial influence enters the agent from what harm it ultimately causes, and instantiate it as a scenario-aligned task suite spanning realistic deployment workflows. Second, a high-fidelity real executable environment: rather than static Q&A or mocked tool responses, agents interact with real tools against dynamically provisioned task artifacts, with persistent state across multi-step interactions that naturally supports long-horizon attack evaluation. Third, trajectory-grounded multi-dimensional evaluation: evaluation consumes the full agent trajectory rather than the reply text or a single tool call, enabling decomposed scoring along three orthogonal dimensions, Outcome Safety, Security Awareness, and Task Utility. We evaluate a broad set of frontier models on AgentCanary against multiple established adversarial attack methods across three agent frameworks. The results reveal that current agents often fail to recognize the attacks they face, particularly under compromised skills, persistent state, and long-horizon execution attacks, and provide a systematic baseline for developing more reliable and secure agent systems.

## 1. 论文的核心问题与整体含义

- **研究动机**：自主AI代理（Autonomous AI Agents）从被动文本生成转向主动数字执行，能够自主调用工具、操作文件系统、持久化状态并执行多步工作流。这种转变将安全威胁从“文本欺骗”升级为“系统入侵”（system compromise）。现有安全评估存在三大瓶颈：
  1. **风险覆盖碎片化**：提示注入、工具投毒、记忆污染等常被孤立处理，缺乏统一抽象。
  2. **执行环境静态/低保真**：依赖静态QA或模拟响应，无法复现真实工具调用链、动态状态变化和持续状态，无法评估长期攻击。
  3. **评估指标单维粗粒度**：仅依赖最终回复或单一工具调用进行二元判断，将结果安全、安全意识和任务效用混为一谈。
- **整体含义**：需要建立面向系统级安全评估的框架，在真实执行环境中测试代理面对多种攻击入口时的完整行为链，从而为安全部署提供可靠基线。

### 核心思想

## 6. 论文的主要结论与发现

1. **安全能力高度不均**：仅Claude Opus 4.6表现突出（总分83.9），第二梯队（Qwen 3.6-Plus 69.7, GPT-5.4 68.5）差距明显；弱模型总分低至37.9（GLM-4.7-Flash）。
2. **安全不遵循简单缩放定律**：同一家族内模型间差异大（如Claude Opus 4.6 vs Sonnet 4.5 差22分），参数规模不能单独解释安全水平。
3. **结果安全与安全意识可解耦**：IPI任务中许多模型OSS很高（>90）但SAS很低（<50），出现“沉默服从”——表面安全但未识别攻击，易被变种击破。
4. **静态评估低估风险**：动态攻击进化（DAE）和技能伪装显著降低多数模型的表现，部分模型安全得分下降超过20分。
5. **持久状态和长地平线攻击最危险**：记忆污染（MC）和长地平线渐进攻击（LPA）的不安全结果率最高（均值42%和51%），显示跨会话推理是当前薄弱环节。
6. **安全与效用正相关**：安全得分高的模型通常任务效用也高（r=0.94），而非简单的安全-效用权衡。
7. **框架选择影响安全**：同一模型在不同框架上得分相差高达15.2分（DeepSeek-V4-Pro），无框架绝对优。
8. **运行时防御有边际收益**：ClawKeeper和Shield平均提升2-8分，但无法替代模型自身安全能力；偶尔出现负收益（SecureClaw）。

## 7. 优点

- **系统性风险组织**：正交风险矩阵使得风险覆盖可审计，便于诊断失败根源。
- **高保真执行环境**：真实工具、持久状态、自动工件预配，贴近实际部署，支持长期攻击评估。
- **多维评估范式**：将结果安全、安全意识、任务效用分开度量，避免了单标签混淆，揭露了“表面安全但未识别”的脆弱性。
- **攻击多样性**：包括静态、动态自适应、长链条、伪装等多种攻击增强，提供更完整的脆弱性刻画。
- **跨框架和防御可扩展**：统一接口促进不同框架和防御的公平对比。
- **开源**：提供代码仓库，可复现。

## 8. 不足与局限

- **框架覆盖不全**：主要结果基于OpenClaw，跨框架实验仅8个开源模型，未对所有12个模型测试框架影响。
- **任务规模有限**：496个种子任务，相对于真实攻击空间较小；未使用大规模自动化生成来覆盖更多边缘情况。
- **评分类依赖LLM**：使用Claude Opus 4.6作为判分器，可能存在判分自身偏差（尽管稳定性较好），且未与其他判分器对比。
- **未报告计算成本**：缺少对基准评估的GPU时间、内存、延迟等资源量化，不利于实际部署决策。
- **防御评估浅层**：仅比较了3种防御的整体得分，未深入分析防御对各个攻击类型的精准拦截/误报率、对任务效用的具体影响。
