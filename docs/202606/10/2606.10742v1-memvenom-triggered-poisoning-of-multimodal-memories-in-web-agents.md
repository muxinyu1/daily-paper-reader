## 摘要
外部记忆已成为现代Web代理的核心组件，通过检索过往经验实现长程推理。然而，这种范式引入了一个关键漏洞：注入记忆的恶意内容可能被持久性地召回，并反复影响代理行为。在这项工作中，我们识别并系统性地研究了多模态记忆投毒——这是Web代理系统中一个被忽视但实际存在的攻击面。我们提出MemVenom，一个统一的黑盒攻击框架，通过协调的文本-图像证据对图结构的外部记忆进行投毒。我们的方法包括两个阶段的设计：（1）触发条件检索攻击，确保恶意记忆的高概率召回；（2）检索后攻击诱导，利用对抗扰动和隐蔽的OCR注入覆盖原始用户目标。与先前针对提示或纯文本记忆的攻击不同，我们的方法无需修改模型参数或重新优化恶意任务即可实现持久、可复用且目标无关的攻击。在多个Web代理框架和视觉语言模型上的实验表明，MemVenom在最小化良性性能影响的前提下实现了强端到端攻击成功率，在GPT-5系列Web代理上高达99.15%，并且能够跨架构和模型规模高效迁移。

## Abstract
External memory has become a core component of modern web agents, enabling long-horizon reasoning through the retrieval of past experiences. However, this paradigm introduces a critical vulnerability: malicious content injected into memory can be persistently recalled and repeatedly influence agent behavior. In this work, we identify and systematically study multimodal memory poisoning, an overlooked yet practical attack surface in web-agent systems. We propose MemVenom, a unified black-box attack framework that poisons graph-structured external memory with coordinated text-image evidence. Our method consists of a two-stage design: (1) a trigger-conditioned retrieval attack that ensures high-probability recall of malicious memory, and (2) a post-retrieval attack induction that leverages adversarial perturbations and stealthy OCR injection to override the original user objective. Unlike prior attacks that operate on prompts or text-only memory, our approach enables persistent, reusable, and goal-agnostic attacks without modifying model parameters or re-optimizing malicious tasks. Experiments across multiple web-agent frameworks and vision-language models demonstrate that MemVenom achieves strong end-to-end attack success with minimal impact on benign performance, reaching up to 99.15% on GPT-5-family web agents, while transferring effectively across architectures and model scales.

- **整体含义**：揭示外部记忆是 Web 代理系统中一个关键的**持久性攻击面**，其威胁超越瞬时的输入操纵，可导致长期、可复用、目标无关的恶意行为。

## 2. 方法论

### 核心思想
MemVenom 是一个统一的两阶段黑盒攻击框架，核心思路是**联合优化恶意记忆被召回的概率**与**召回后诱导代理优先执行恶意行动的能力**。攻击分为三个阶段：

1. **检索阶段优化（Stage 1）**：设计一个视觉触发器（trigger），使携带该触发器的网页截图在嵌入空间中形成紧凑簇，并与良性记忆空间分离，从而极大提高恶意记忆被召回的概率。
2. **后检索阶段攻击（Stage 2）**：通过**对抗扰动**（ℓ∞ 约束）与**隐蔽 OCR 注入**（在基图像上嵌入优先指令文本）构建一个**目标无关的记忆优先化组件**——该组件不绑定具体恶意目标，仅编码“优先遵循召回记忆”的信号，因此可跨不同恶意目标复用。
3. **恶意子图组装（Stage 3）**：将召回导向组件、恶意目标组件、优先化组件通过图边连接为恶意子图，注入干净记忆图。更换恶意目标时只需替换目标组件，无需重新优化触发器和优先化组件。

### 关键技术细节
- **触发器优化**：最小化触发器截图嵌入与簇中心的距离（紧凑性），同时最大化与良性截图及良性记忆节点的距离（分离性）。公式包含三项：簇内紧凑度、与原始观测的分离、与现有记忆空间的分离。
- **记忆优先化组件构建**：使用代理编码器（surrogate encoder）进行黑盒优化；迭代更新桥接图像（bridge image），使其同时贴近优先指令文本且远离当前扰动基图；最终通过对抗扰动和 OCR 注入将优先信号嵌入图像。
- **黑盒假设**：攻击者无法访问 VLM 骨干、检索编码器、策略模型、推理提示或内部评分，只能注入少量多模态记忆条目。

## 6. 主要结论与发现

1. **攻击成功率高**：在 GPT-5.4 上，端到端攻击成功率（ASR-ra）达到 99.15%（Phishing/Redirection 场景），多个配置下 ASR-a 接近 100%。

3. **良性效用保持良好**：投毒后良性任务完成率（PU）与投毒前（BU）相当或略微变化，说明攻击主要在触发条件下激活，不持续干扰正常操作。

5. **现有防御不足**：硬编码规则与 LlamaGuard 仅能部分降低 ASR-ra，无法根除威胁，需专门针对记忆来源与任务对齐的防御。
6. **模块化设计优势**：更改恶意目标只替换目标组件，触发器和优先化组件可复用，显著降低攻击成本。

## 8. 不足与局限

- **实验环境限制**：评估在受控沙盒中进行，真实 Web 代理部署可能涉及更异构的记忆管线、检索策略、动作约束，攻击效果有待验证。
