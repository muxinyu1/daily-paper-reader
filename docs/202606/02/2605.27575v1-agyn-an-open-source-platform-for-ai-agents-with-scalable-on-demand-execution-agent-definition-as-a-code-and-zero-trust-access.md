## 摘要
随着组织转向生产部署AI代理，这些代理执行非确定性工作流、维护有状态会话，并且通常以特权访问内部服务操作，工程挑战从构建单个代理转向以适当的隔离、治理和安全性大规模运行它们。本文介绍了Agyn，一个围绕三个关键原则设计的开源平台，适用于代理工作负载：基于Kubernetes的信号驱动、有状态的无服务器运行时；用于代理和工具链定义的Terraform提供程序；以及基于零信任和最小权限原则的安全模型。Agyn是代理无关、模型无关和云无关的。

## Abstract
As organizations move toward production deployments of AI agents, which execute non-deterministic workflows, maintain stateful sessions, and often operate with privileged access to internal services, the engineering challenge shifts from building individual agents to operating them at scale with proper isolation, governance, and security. In this paper we present Agyn, an open-source platform designed around three key principles tailored for agent workloads: a signal-driven, stateful serverless runtime on Kubernetes; a Terraform provider for agent and harness definition; and a security model grounded in zero-trust and least-privilege principles. Agyn is agent-agnostic, model-agnostic, and cloud-agnostic.

# Agyn: 一个开源AI代理平台——详细总结

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：组织将AI Agent投入生产时，面临三大运营挑战：① 非确定性工作流与有状态会话需要高效、弹性的计算资源，但预先预留资源会导致大量空闲浪费；② 代理定义（提示词、工具、密钥、资源限制等）是重要的生产决策，应像基础设施一样进行版本控制、代码审查和可回滚；③ 代理需要以身份访问内部服务，但传统安全模型假设网络内部是可信的，而LLM易受提示注入攻击，直接向模型暴露凭证风险极高。

- **现有系统不足**：Serverless平台（如Knative、AWS Lambda）缺乏对有状态会话和代理触发器的原生支持；基础设施即代码（IaC）框架缺乏面向代理工作负载的抽象层；零信任网络系统要求手动管理身份生命周期，与临时性代理执行不匹配。目前没有任何平台能同时满足上述三个需求。

- **整体含义**：Agyn提供了一个开放、可自托管的平台，将“每个代理定义一次、可复用、按需实例化”的模式与严格的安全治理相结合，旨在解决AI Agent规模化运营中的隔离、治理与信任难题。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 平台围绕三个支柱设计：**信号驱动的有状态无服务器运行时**、**代理定义即代码**（通过Terraform）、**零信任最小权限安全**。
- 代理定义集中管理，每个运行实例绑定到单个用户和单个任务（如一个对话线程），实例相互隔离。

### 关键技术细节

#### ① 运行时：信号驱动的有状态Serverless
- **信号驱动生成**：用户在线程上发送消息 → Notifications服务发布事件 → Agents Orchestrator订阅事件 → 解析代理定义中的密钥、请求OpenZiti身份、通过k8s-runner创建Kubernetes Pod（挂载持久卷，恢复之前会话状态）。
- **空闲回收**：活跃时每10秒发送心跳；空闲超时（默认5分钟）后停止Pod并删除网络身份，但持久卷和会话历史保留以备下次恢复。
- **与其他函数即服务(FaaS)的区别**：传统FaaS针对HTTP请求设计，而代理工作负载的触发器是对话消息，工作单元持续数分钟，状态必须在多次生成之间存活。Agyn实现“临时计算+持久状态”模式。

#### ② 配置：代理与工具链即代码
- **Terraform提供程序**：声明式定义代理容器、工作空间配置、系统提示、模型、密钥、MCP服务器、持久卷等全部周边基础设施（harness）。支持内置三种预构建代理镜像（Claude Code、OpenAI Codex、Agyn自己的循环实现），也可自定义容器。
- **模块化复用**：Terraform的模块结构允许将常见工具链模式（如带密钥的MCP）打包成共享模块，跨代理复用，确保一致性。
- **生效方式**：`terraform apply`后，定义通过Gateway解析到Agents Service，后续消息自动产出新Pod，无需重启。

#### ③ 安全：零信任与最小权限
- **每容器隔离**：主容器运行代理进程，Sidecar容器运行MCP服务器。各容器拥有独立文件系统和进程树，仅共享回环接口。密钥仅注入需要的容器（通常是MCP Sidecar），绝不注入驱动LLM的代理容器。
- **零信任覆盖网络**：使用OpenZiti：每个代理在生成时获得临时x509身份，mTLS在应用代码之前建立身份验证。Gateway提取身份并传播到整个调用链，每个下游服务知晓请求发起的代理。基于属性的访问控制（ABAC）策略限制每个代理只能访问明确允许的服务。
- **基于关系的授权（ReBAC）**：底层使用OpenFGA（受Zanzibar启发）通过图遍历实施细粒度权限。代理角色（所有者、维护者、参与者）限定谁可以配置或交互；线程访问限定为参与者。

### 实现细节
- 平台服务用Go编写，PostgreSQL作为持久存储，Redis用于发布/订阅事件分发，OpenZiti Go SDK嵌入基础设施服务（避免Istio Sidecar冲突），代理Pod使用Ziti TPROXY Sidecar实现透明覆盖网络。
- 三种身份生命周期共存：临时每工作负载代理身份、临时每Pod服务身份、持久运行/应用/设备身份。Ziti管理服务封装控制器API调用并运行基于租约的垃圾回收。
- 完整栈打包为Helm Chart，可在任何Kubernetes集群上部署；提供一键式引导仓库。

- **Agyn是唯一同时满足以下所有条件的平台**：自托管、提供预构建Agent镜像、支持模型上下文协议（MCP）隔离、声明式配置（Terraform）、Serverless自动扩缩、凭证隔离（防止密钥泄露给LLM）、零信任网络架构。

## 7. 优点

1. **设计整合度高**：将Serverless弹性、IaC治理和零信任安全三个独立领域的技术有机融合，形成一个统一的平台方案。
