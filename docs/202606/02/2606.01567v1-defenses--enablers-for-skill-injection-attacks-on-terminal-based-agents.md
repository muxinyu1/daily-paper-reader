## 摘要
大型语言模型（LLM）代理越来越依赖可重复使用的技能，即描述特定任务流程的文档。然而，这为代理引入了新的攻击面。我们针对这一威胁研究了两个互补方向。首先，我们评估了基于守护者的防御：一个作为技能文件访问中介的中间LLM代理（动态守护者），或在构建时预先重写这些文件（静态守护者）。在三个LLM代理家族中，我们的守护者将攻击成功率（ASR）降低了一半以上，同时保持了任务效用。其次，我们通过四种保留恶意指令但改变措辞的攻击，对它们进行了攻击重新框架的压力测试。对于非守护者设置，重新框架将ASR推高至81.4%，但动态守护者将其降至18.6%，表明实时中介是一种稳健的防御措施。

## Abstract
Large language model (LLM) agents increasingly rely on reusable skills i.e. documents describing task-specific procedures. However, this introduces a new attack surface for agents to manage. We study two complementary directions for this threat. First, we evaluate guardian-based defenses: an intermediary LLM agent that acts as a mediator for skill file access (dynamic guardian) or pre-rewrites these files at build time (static guardian). Across three LLM agent families, our guardians cut attack success rate (ASR) by well over half while preserving task utility. Second, we stress test them through attack reframing using four attacks that preserve the malicious instruction but change the phrasing. For non-guardian setup, the reframing pushes the ASR up to 81.4\%, but the dynamic guardian brings it down to 18.6\%, showing that real-time mediation is a robust defense.

# 论文总结

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）代理越来越多地使用可复用的“技能”文件（如 `.md` 文档）来描述任务流程，但这些技能可从公开市场获取，引入新的攻击面。攻击者可修改技能文件，诱导代理执行非预期恶意操作（如数据泄露、文件删除、权限提升等）。
- **整体含义**：现有代理缺乏对技能信任边界的维护。本文旨在构建有效的防御机制（守护者子代理）来过滤恶意注入，同时探索攻击者如何通过重新措辞（reframing）绕过检测，从而全面评估威胁并验证防御鲁棒性。

## 2. 方法论：核心思想与关键技术细节

- **核心思想**：在代理与技能生态之间插入一个守护者LLM子代理，负责过滤或重写技能文件中的恶意内容，防止注入抵达主代理。
- **两种守护者变体**：
  - **动态守护者（Dynamic Guardian）**：作为MCP（Model Context Protocol）服务器运行，在运行时拦截每次技能文件访问，经守护者LLM处理后返回清理后的内容（实时过滤）。
  - **静态守护者（Static Guardian）**：在构建时（任务开始前）扫描所有技能文件，通过守护者LLM重写文件，移除注入，之后代理运行时无需额外开销。
- **攻击重构（Attack Reframing）**：在保留恶意指令语义的前提下，改变其表述形式以增加攻击覆盖率。包括四种策略：
  1. **翻译攻击**：将整个技能文件（含注入）翻译成西班牙语或日语。
  2. **盲引用（Blind Reference）**：将注入包装为 YAML 配置 `precondition`，使代理自动加载。
  3. **交叉引用（Cross Reference）**：引用不存在的 `SCOPE.md` 文件，要求代理将其指令与当前指令视为同等约束。
  4. **描述陷阱（Description Trap）**：创建一个虚构的“通用助手”技能，其宽泛描述导致自动加载，注入藏于其中。

## 6. 主要结论与发现

- **守护者防御有效**：动态和静态守护者均将攻击成功率（ASR）降低超过一半。例如，Sonnet-4.5+CC 的 ASR 从 36.0%（Vanilla）降至 12.9%（Dynamic）和 7.2%（Static），任务成功率（TSR）基本保持或略有提升。
- **攻击重构放大威胁**：对非守护者设置，四种重构联合将 ASR 从原始 55.8% 提升至 81.4%，表明原基准低估了真实威胁。
- **动态守护者最鲁棒**：面对全部重构攻击，动态守护者仍将 ASR 降至 18.6%，显著低于静态的 39.5%，证明实时中介是更强防御。
- **系统提示防御不足**：即使有针对性的 Oracle 警告（SysTargeted）也只能将 ASR 降至 23.0%，远不及守护者效果，说明内容过滤优于依赖代理自我约束。

## 7. 优点

- **方法创新**：首次系统提出基于守护者子代理的防御架构（动态MCP与静态重写），并验证其在多种模型上的有效性。
- **攻击重构全面性**：不仅评估原始注入，还通过翻译和自动发现的新变种模拟了攻击者可能使用的更隐蔽手段，使威胁评估更贴近真实场景。
- **实验设计严谨**：在统一基准上对比多种防御方式（无防御、提示防御、两种守护者），并测量ASR和TSR双重指标，确保公平。
- **消融分析深入**：分析了动态守护者交互轮次与ASR的关系，指出单次大块请求是薄弱环节；静态守护者内容变换特征（如复杂技能内容丢失、注入“洗白”）。
- **负责任披露**：代码开源，实验在隔离沙箱中完成，遵循双用研究伦理。

## 8. 不足与局限

- **基准覆盖有限**：仅使用 Skill-Inject 基准（48个注入），真实市场有数千技能（Liu et al., 2026），其他攻击类型（依赖混淆、沙箱逃逸、供应链攻击）未涉及。
- **模型与框架覆盖**：仅测试3个模型族和2个代理框架（Claude Code、OpenCode），其他推理设置或小模型可能表现不同。
- **守护者模型耦合**：实验中守护者与主代理使用相同基础模型，未研究异构守护者（如专用安全模型）的效果。
- **攻击重构评估子集**：重构实验仅在43对子集上报告，绝对ASR不能直接与主实验139沙箱对比；风险来自自动发现迭代仅50轮，可能遗漏更多变种。
- **裁判可靠性**：任务成功率依赖LLM裁判（Sonnet 4.5），尤其静态重写后技能差异大时可能有噪声。
- **语言与领域局限**：任务和注入主要为英文，重构仅测试西班牙语和日语，其他语言或非终端领域（GUI、浏览器）未验证。
- **算力资源未详细量化**：缺少具体GPU总耗时、推理资源消耗，不利于复现和比较成本。
