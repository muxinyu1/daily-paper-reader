## 摘要
企业入侵响应仍然依赖于静态剧本和分析师驱动的分类，导致从警报生成到遏制之间存在延迟。我们提出了Agentra，一种可监督的多智能体入侵响应系统（IRS）框架，该框架将来自IDS、EDR和XDR平台的警报转化为基于MITRE ATT&CK、MITRE D3FEND和NIST CSF 2.0的结构化事件响应计划。Agentra通过角色限定的智能体分解响应推理，通过有界的规划器-验证器审查循环验证提出的计划，通过审核员安全网关筛选检索到的威胁情报，通过行动目录和风险评分筛选行动，并将决策记录在仅追加的审计日志中。我们基于来自ThreatHunter-Playbook、Splunk BOTSv3和DARPA OpTC的120个事件语料库，将Agentra与静态OASIS CACAO v2.0网络剧本基线进行了比较。最强配置将考虑误报的IRS F1分数从0.61提高到0.84，并在仅规划器配置引入不安全过度反应后，将预测的有害行动率恢复到0.0%的静态基线水平。这些结果表明，多智能体响应规划可以在保留分析师批准和可审计性的同时，提高基于本体的IRS覆盖率。

## Abstract
Enterprise intrusion response still depends on static playbooks and analyst-driven triage, creating delay between alert generation and containment. We present Agentra, a supervisable multi-agent Intrusion Response System (IRS) framework that converts alerts from IDS, EDR, and XDR platforms into structured incident response plans grounded in MITRE ATT&CK, MITRE D3FEND, and NIST CSF 2.0. Agentra decomposes response reasoning across role-scoped agents, validates proposed plans through a bounded Planner--Validator review loop, screens retrieved threat intelligence through a Moderator security gateway, gates actions through an Action Catalog and risk score, and records decisions in an append-only audit log. We evaluate Agentra against a static OASIS CACAO v2.0 cyber-playbook baseline on a 120-event corpus drawn from ThreatHunter-Playbook, Splunk BOTSv3, and DARPA OpTC. The strongest configuration improves FP-aware IRS F1 from 0.61 to 0.84 and restores the projected harmful-action rate to the static baseline level of 0.0% after Planner-only configurations introduce unsafe overreaction. These results indicate that multi-agent response planning can improve ontology-grounded IRS coverage while preserving analyst approval and auditability.

### 1. 核心问题与整体含义（研究动机和背景）

- **问题**：企业入侵响应严重依赖静态剧本（playbook）和安全分析师手动分类（triage），导致从告警生成到采取遏制行动之间存在显著延迟。
- **背景**：检测速度已大幅提升（CrowdStrike 2025年报告平均eCrime突破时间29分钟，最快27秒；Unit 42报告最快四分之一入侵在1.2小时内达到数据外泄），但响应仍以人工为主。IBM报告平均漏洞生命周期241天，美国平均违约成本1022万美元。现有SOAR系统依赖确定性、分析师维护的工作流，难以适应多变的攻击战术和跨平台攻击链。
- **整体含义**：需要一种能够自动生成响应计划但仍保持人类监督和可审计性的框架。该框架应基于安全本体（MITRE ATT&CK, D3FEND, NIST CSF 2.0），并处理威胁情报检索中的间接提示注入、模型输出不可信等安全问题。

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将入侵响应构建为**可监督的多智能体规划**，而非直接自主执行。通过角色分离、分阶段审查、受限行动目录和可审计日志，确保最终响应计划既有效又安全。
- **架构（Agentra 框架）**：
  - **Incident Context 构建器**：将异构告警证据归一化为结构化记录（包含IoC、根因假设、时间线、ATT&CK技术、D3FEND候选缓解措施、NIST CSF功能等）。
  - **Planner（规划器）**：基于上下文和已筛选的知识，生成候选响应计划（IRP），序列化为OASIS CACAO v2.0剧本。规划器无执行权限。
  - **Validator（验证器）**：独立审查规划器提出的计划，返回批准/拒绝及理由。实现有界的**规划器-验证器审查循环**：被拒绝的计划可返回规划器修改最多R轮；若未达成共识，升级给分析师。
  - **Moderator（审核员）**：作为安全网关，负责检索内部知识（通过Graph-RAG）和外部威胁情报，并对检索内容进行安全检查（如过滤指令注入、允许列表违规、失范缓解标识等），然后将净化的上下文提供给规划器。规划器从不直接接触原始检索内容。
  - **Executor（执行器）**：仅在分析师批准后，通过**行动目录**（Action Catalog）和**风险评分**（结合资产关键性、爆炸半径、可逆性）逐步执行。包含**理智检查器**（检查内部一致性和状态变化）和**风险分层**（建议/批准/高级批准）。
  - **审计日志**：采用仅追加（append-only）的Merkle树结构，记录所有提示、检索、验证投票、风险决策、分析师批准和执行动作，确保可审计性和防篡改。
- **形式化流程（Algorithm 1）**：
  1. 输入告警证据E → 构建事件上下文I，提取检索键K。
  2. 通过Graph-RAG和外部检索获取Xint和Xext，经Moderator筛选后得X。
  3. Planner根据I、X和已有剧本库生成候选计划P。
  4. 在最多R轮内，Validator池（至少k个通过）对P投票；若未达阈值则升级。
  5. 分析师审批计划；检查状态是否变化（若变化则无效重新分析）。
  6. 对每个动作a检查是否在行动目录中、通过理智检查、计算风险分数，按授权层级执行。
  7. 所有步骤记录到审计日志。

### 6. 论文的主要结论与发现

- **核心结果**：全栈Agentra（A4, M1组合）将FP-aware IRS F1从静态CACAO基线的0.61提升到**0.84**；预测有害行动率从规划器单独配置（A1）的14–16%**恢复到0.0%**（与CACAO基线持平）。
- **组件贡献分解**：
  - A1（仅Planner）：召回率大幅提升（M4达0.90），但D3FEND精度崩塌（≤0.15），有害动作率上升（14–16%），误报率高（M4达0.97）。
  - A2（Validator）：依赖组合效应；对M1降低误报但未改善质量；对M5几乎崩溃召回（0.14），因验证器过于严格且超时。
  - **A3（Moderator+CTI）是质量转折点**：D3FEND精度跳升（M1 0.23→0.79），有害动作率降至≤0.9%，召回恢复。
  - A4（Graph-RAG）进一步稳固，但不同组合间存在权衡：M4召回完美（1.00）但误报高（0.90），F1=0.81；M1达到最高F1=0.84；M5 D3FEND精度0.95但召回仅0.57。
- **replace-CACAO vs. enhance-CACAO**：当有强大的本地规划器时，replace-CACAO（M1）略优于enhance-CACAO（M2），因A3/A4的接地弥补了缺失的起始点。
- **总体**：多智能体响应规划可在保持分析师审核和可审计性的同时，提高基于本体的入侵响应覆盖率。但最终质量高度依赖模型行为和组件组合。

### 7. 优点

- **架构设计**：角色分离（规划、验证、审核、执行、审计）提供了多层监督，防止单点失败。**可监督性**是亮点，不同于完全自主的代理系统。
- **安全性考虑**：明确将检索内容和模型输出视为不可信，引入Moderator审核、行动目录、风险评分、状态变化检查，针对间接提示注入等威胁有防御设计。
- **审计追溯**：仅追加的Merkle树审计日志，保证决策记录的完整性。
- **评价指标全面**：不仅考虑召回，还强调误报（FP-aware F1）和有害动作率，更贴近实际运营需求。
- **消融实验严谨**：逐步叠加组件，清晰展示每个模块的边际贡献。

### 8. 不足与局限

- **统计可靠性缺乏**：仅报告单种子点估计，无置信区间，无法判断观察到的差异是否统计显著。
- **验证器多样性缺失**：实验中仅使用单验证器，无法体现k-of-n异构池的故障容错优势。且验证器与规划器可能共享相同基础模型，独立性不足。
- **对抗性评估缺失**：Moderator的提示注入和检索投毒防御未在测试集中专门验证（论文承认）。
- **生态效度有限**：使用静态公开数据集，而非实时网络靶场或真实企业环境；操作安全性通过投影而非实际执行测量。
- **模型记忆风险**：LLM在训练数据中可能见过这些事件，匿名化不能完全消除。
- **延迟不适用实时**：平均40秒/事件，仅适合批量分类，不满足秒级响应。
- **D3FEND标签为人工注释**：依赖分析师判断，非绝对标准。
- **可复现性**：虽然使用了公开数据集和模型，但具体版本（如Kit 的CACAO映射、缓存时间点）可能影响结果。论文提到评估时固定了本体和框架工件的缓存版本。
