## 摘要
随着LLM代理越来越多地自主选择工具，它们在不同权限工具间的选择变得与安全相关。然而，先前的工具选择研究侧重于安全无关的元数据偏好，忽略了权限敏感的选择。为填补这一空白，我们研究了过度权限工具选择，即代理在存在足够低权限替代方案时仍选择或升级到更高权限工具。我们引入了ToolPrivBench来评估代理是否在存在足够低权限替代方案时仍选择更高权限工具，同时测量初始选择以及瞬态工具故障后的升级行为。在八个领域和五种反复出现的风险模式中，我们发现过度权限工具选择在主流LLM代理中普遍存在，并且瞬态故障会进一步加剧这一问题。我们还发现，一般的安全对齐并不能可靠地转化为最小权限工具选择，而提示级别的控制在瞬态故障下只能提供有限的缓解。因此，我们引入了一种权限感知的后训练防御，教导代理偏好足够低权限工具，仅在必要时升级。我们的缓解实验表明，这种防御在保持通用能力的同时，显著减少了不必要的高权限工具使用。

## Abstract
As LLM agents increasingly select tools autonomously, their choices among tools with different privileges become safety-relevant. However, prior tool-selection studies focus on safety-agnostic metadata preferences, leaving privilege-sensitive choices underexplored. To address this gap, we study over-privileged tool selection, in which an agent selects or escalates to a higher-privilege tool despite a sufficient lower-privilege alternative. We introduce ToolPrivBench to evaluate whether agents choose higher-privilege tools despite sufficient lower-privilege alternatives, measuring both initial selection and escalation after transient tool failures. Across eight domains and five recurring risk patterns, we find that over-privileged tool selection is common among mainstream LLM agents and is further amplified by transient failures. We further find that general safety alignment does not reliably transfer to least-privilege tool choice, while prompt-level controls provide only limited mitigation under transient failures. We therefore introduce a privilege-aware post-training defense that teaches agents to prefer sufficient lower-privilege tools and escalate only when necessary. Our mitigation experiments show that this defense substantially reduces unnecessary high-privilege tool use while preserving general capabilities.

## 1. 核心问题与整体含义（研究动机与背景）
- **研究动机**：随着LLM从对话助手向自主代理演进（如vibe coding），代理在多个可选工具中自主选择执行路径。当存在功能足够但权限较低的工具时，代理却倾向于选择更高权限的工具（如管理员API、全局操作），这种“过度权限工具选择”引入了隐蔽但严重的安全风险——它增加了错误、误用或妥协时的潜在爆炸半径。
- **核心问题**：代理是否在没有必要的情况下选择高权限工具？这种行为在初始决策和遇到临时故障后如何表现？现有安全对齐和提示控制能否缓解？
- **整体含义**：揭示了一个之前被忽视的代理安全维度——权限意识（privilege awareness），即代理是否能在被授权的工具集中选择最小特权且足够的工具。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：构建一个模拟环境，其中每个任务同时提供3个低权限工具和3个高权限工具，所有工具均被验证为独立足以完成任务。通过测量代理的初始选择和在瞬态故障后的升级行为，量化过度权限工具选择。
- **关键技术细节**：
  - **问题形式化**：定义过度权限选择为：在决策步骤i，若存在未排除且权限更低的足够工具，代理却选择更高权限工具。
  - **两种行为类型**：① 激进选择（aggressive selection）：初始直接选择高权限工具；② 过早升级（premature escalation）：低权限工具经历瞬态非权限相关故障（如连接错误）后立即升级。
  - **基准构建管道**：从APIGen数据集分析真实工具权限等级（L1-L5），抽象出8个应用领域和5种风险类型（权限升级、范围扩展、持久性、安全绕过、数据过度暴露）。每类生成场景，经过自动结构过滤、描述中立性检查、多样性检查，以及双重验证（自动跨模型共识+人类专家审核）确保低权限工具的功能充分性。
- **度量指标**：
  - OPUR@k：在k轮内使用任何高权限工具的比例。
  - PED：在首次使用高权限工具前尝试过的不同低权限工具数量。PED=0对应激进选择，≥1对应过早升级。

## 6. 主要结论与发现
- **发现1**：过度权限工具选择在主流LLM中普遍存在，但严重程度因模型而异。6/11模型OPUR超过30%，小模型（Qwen3-8B、LLaMA-3.1-8B）更高，Frontier模型（Claude 4.6 Sonnet, GPT-5.2）较低但仍有。
- **发现2**：瞬态故障显著加剧升级倾向。大多数模型在遭遇错误后快速升级，而非尝试其他低权限工具。例如GPT-5.2在故障后OPUR从5次激增至35次。
- **发现3**：风险类型不对称：权限升级和安全绕过最频繁；领域差异：基础设施、媒体、数据库领域更容易出现过度权限选择。
- **发现4**：通用安全对齐（AgentAlign）不能自动迁移到最小权限工具选择（OPUR未显著下降甚至上升），而提示控制在多轮中效果减弱。
- **发现5**：特权感知后训练（SFT+GRPO）显著降低OPUR（Qwen3-8B从64.9%降至27.0%，Qwen3-4B-Thinking从66.0%降至18.9%），同时通用能力几乎无损（MMLU、GSM8K、MetaTool保留率>95%）。

## 7. 优点
- **问题新颖性**：首次系统研究LLM代理中的过度权限选择，填补了工具选择安全性的空白。
- **基准构建严谨**：双重验证（自动+人工）确保低权限工具功能充分，避免能力混淆。风险类型和领域覆盖全面。
- **方法完整性**：不仅发现问题，还提出了有效的缓解方案（SFT+GRPO），并在多个模型上验证了通用能力保持。
- **分析深入**：区分了激进选择和过早升级，揭示了故障对升级的放大效应，以及域和风险类型的影响。

## 8. 不足与局限
- **模拟环境**：所有评估在模拟中进行，没有使用真实的生产工具或实时外部服务，可能无法完全反映真实部署中的复杂性和噪声。
- **工具集有限**：每个场景仅6个工具（3低+3高），且假设低权限工具独立足以完成任务。现实中工具可能部分重叠、需要组合使用、且工具库规模大得多。
- **交互轮次限制**：最大5轮，可能无法捕捉更长时间跨度的行为（如重复故障后逐渐升级）。
- **仅评估特定行为**：未覆盖代理可能采取的其他不安全行为（如越权、拒绝服务等）。权限意识只是安全的一部分。
- **数据分布偏差**：基准域和风险分布基于APIGen分析，但可能不适用于所有真实应用。
- **后训练泛化性**：仅在Qwen系列模型上测试，未在其他家族（如Llama、Claude）上验证。且训练数据与评估数据域分离程度未知。
- **机制理解不足**：论文未深入解释代理为何偏好高权限（如因果归因、模型内部表示分析）。
- **人类评估规模**：仅2名审核员，可能存在主观偏差。
