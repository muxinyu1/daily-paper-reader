## 摘要
提出一种协议驱动框架，将SBOM和AIBOM工件绑定到确定性环境捕获和结构化运行时遥测中。可利用性根据声明的工件、观察到的激活条件和强制执行的策略来计算。CSAF VEX通报由静态和运行时证据组合生成，经过加密签名，并通过确定性重放进行验证。评估使用约10000个组件条目，跨越合成自主AI工作负载（50到5000个组件），并结合了OSV、GitHub Advisory、KEV和EPSS数据集。

## Abstract
A protocol driven framework is presented that binds SBOM and AIBOM artefacts to deterministic environment capture and structured runtime telemetry. Exploitability is computed from declared artefacts, observed activation conditions, and enforced execution policies. CSAF VEX advisories are generated from combined static and runtime evidence, cryptographically signed, and validated through deterministic replay. Evaluation uses approximately 10000 component entries across synthetic Agentic AI workloads 50 to 5000 components, incorporating OSV, GitHub Advisory, KEV, and EPSS datasets.

## 论文详细总结（自动生成）

### 1. 论文的核心问题与整体含义
- **研究动机**：自主AI系统（Agentic AI）由基础模型、提示模板、工具连接器、编排逻辑和容器化依赖组成，其可利用性无法仅从静态软件物料清单（SBOM）推断。现有的通用安全咨询框架（CSAF）/漏洞可利用性交换（VEX）工作流基于静态组件-CVE关联，缺乏运行时验证，导致大量误报和低效的修复优先级。
- **整体目标**：提出一个协议驱动的自动化框架，将人工智能物料清单（AIBOM）与确定性环境捕获和运行时遥测绑定，从声明式披露转变为“执行约束”的可利用性评估，生成可复现、加密可验证的CSAF-VEX通报。

### 2. 方法论
- **核心思想**：可利用性取决于三个条件的合取——（1）存在性：易受攻击的组件在SBOM/AIBOM中声明；（2）激活：组件在运行时被实际调用；（3）可行性：环境策略（网络出口、权限、隔离）允许利用。只有三者同时满足才视为“受影响”。
- **关键技术架构**：三种协议协同工作：
  - **MCP（模型上下文协议）**：执行前捕获确定性环境包（容器摘要、模型哈希、提示模板提交ID、工具端点、SBOM清单、安全策略），加密签名作为来源锚点。
  - **A2A（代理间协议）**：运行时部署侧车代理收集遥测数据（系统调用、动态库加载、网络连接、工具调用、权限），所有事件时间戳、哈希、签名。
  - **AGNTCY（治理层）**：协调代理注册、策略执行、CSAF-VEX生成、加密签名、透明度日志记录。
- **流程**（6个阶段）：
  1. 环境初始化与MCP捕获 → 2. 执行工作量并收集A2A遥测 → 3. 使用OSV、GitHub Advisory、KEV、EPSS进行组件-CVE匹配 → 4. 基于激活条件和策略约束的可利用性推理 → 5. CSAF-VEX通报生成（包含CVE ID、组件哈希、AIBOM引用、运行时证据哈希、来源引用）→ 6. 确定性重放验证（相同条件下重新执行，比较哈希一致性）。
- **部分伪代码**：论文提供了30行伪代码，清晰封装了上述6阶段逻辑。
