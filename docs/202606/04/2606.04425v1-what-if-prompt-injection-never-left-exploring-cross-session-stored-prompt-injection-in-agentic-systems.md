## 摘要
现代智能体系统将大型语言模型从会话绑定的助手转变为有状态系统，这些系统通过记忆、文件系统、工具和其他长期存在的上下文工件在会话间持久化和演化共享的世界状态。这一转变从根本上扩展了提示注入的攻击面。然而，先前关于提示注入的工作主要集中于单次会话内的模型级威胁，忽视了跨会话持久化系统状态如何根本性地改变智能体系统的系统级风险。受网络系统中持久化跨站脚本的启发，我们引入了跨会话持久化提示注入，其中一次成功的注入可以持久存在于智能体系统状态中，并在原始攻击者交互结束后长时间静默地影响未来的执行。为了系统性地研究这一威胁，我们形式化了持久化提示注入，并开发了一个关于对抗性内容如何跨会话持久化并影响智能体系统的分类法。我们进一步开发了一个基准测试和沙盒工具包，用于评估持久化提示注入的风险，从而能够对跨不同模型、攻击目标和持久化渠道的攻击成功进行定量分析。我们的发现强调，持久化将提示注入从一种短暂的模型级威胁转变为嵌入在智能体执行状态中的长期系统级漏洞。我们希望这项工作能引起对这一新兴威胁的更广泛关注，并激励社区系统性地研究和缓解智能体系统中由持久化引起的系统风险。

## Abstract
Modern agentic systems transform LLMs from session-bounded assistants into stateful systems that persist and evolve shared world state across sessions through memories, filesystems, tools, and other long-lived contextual artifacts. This shift fundamentally expands the attack surface of prompt injection. However, prior works on prompt injection have largely focused on model-level threats within a single session, overlooking how cross-session persistent system state fundamentally changes the system-level risk of agentic systems. Inspired by stored cross-site scripting in web systems, we introduce cross-session stored prompt injection, where a successful injection can persist within agentic system state and silently influence future executions long after the original attacker interaction has ended. To systematically study this threat, we formalize stored prompt injection and develop a taxonomy of how adversarial content persists and affects agentic systems across sessions. We further develop a benchmark and sandbox toolkit to evaluate the risks of stored prompt injection, enabling quantitative analysis of attack success across different models, attack goals, and persistence channels. Our findings highlight that persistence transforms prompt injection from an ephemeral model-level threat into a long-lived system-level vulnerability embedded within agent execution state. We hope this work draws broader attention to this emerging threat and motivates the community to systematically study and mitigate system risks arising from persistence in agentic systems.

- **核心问题**：现代智能体系统（agentic systems）将大语言模型（LLM）从单会话绑定的助手转变为跨会话持久化状态系统（通过记忆、文件系统、工具等长期存在的上下文工件）。这种转变使得传统的提示注入（prompt injection）威胁从单次会话内的短暂模型级漏洞演变为跨会话的持久系统级漏洞，而现有研究对此缺乏系统性认识。
- **研究动机**：现有提示注入研究主要关注单次会话内的威胁（直接/间接提示注入），假设恶意指令随会话终止或上下文清空而消失。然而，当对抗性内容可以写入持久化系统状态（如记忆、配置文件、工具可见状态）并在未来会话中被重新纳入执行上下文时，注入和利用在时间上解耦，形成了类似于Web系统中存储型XSS的威胁。
- **整体含义**：本文首次识别并形式化了**跨会话存储提示注入（Stored Prompt Injection, SPI）**，将其定义为智能体系统中的一类基础性系统级漏洞，强调持久化状态管理是新兴的安全边界，需要社区系统性地研究和缓解。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将SPI建模为两阶段攻击：（1）**注入阶段**：攻击者通过用户输入、外部消费内容、供应链工具等渠道，将恶意内容写入持久化上下文通道；（2）**激活阶段**：在后续会话中，恶意内容被上下文构建函数重新纳入代理上下文，影响模型行为。攻击成功取决于三个耦合问题：不安全持久写入、恶意下游重新激活。
- **关键技术与形式化**：
  - **系统模型**：将智能体系统视为上下文构建流水线（context construction pipeline），输入由多个上下文通道组装：当前用户查询 \(q_t\)、指令上下文 \(I\)、会话历史 \(H_t\)、工具接口上下文 \(T\)、检索上下文 \(R_t\)、文件支撑上下文 \(S_t\)。模型处理组装后的输入 \(x_t = \mathcal{A}(q_t, I, H_t, T, R_t, S_t)\)。
  - **持久化上下文分类**：根据持久性和重新纳入可能性分为三类：
    - **临时（ephemeral）**：会话绑定，结束时丢弃（如会话历史）。
    - **条件持久（conditionally persistent）**：持久存储但仅在有条件时被纳入（如档案记忆、工具响应结果）。
    - **强持久（strongly persistent）**：跨会话默认或高度可能被纳入（如系统提示、`AGENTS.md`、工作记忆、工具描述）。
  - **威胁模型**：攻击者通过**写原语** \(w: u \mapsto p \in \mathcal{P}\) 将外部内容写入持久化通道，意图长期影响未来代理行为。攻击者不能直接修改代理逻辑或强制上下文包含，只能影响持久状态。
  - **SPI定义**：当对抗性内容 \(p\) 被写入或修改持久化上下文 \(\mathcal{P}\)，并在后续执行中被纳入上下文 \(x_t = \mathcal{A}(q_t, \cdots, \mathcal{P})\) 且被模型解释为指令，从而改变预期行为时，即构成SPI攻击。
- **分类法三维度**：
  1. **注入源**：用户提供内容、外部消费内容、供应链可安装工具。
  2. **持久化通道**：代理记忆（工作/档案记忆）、工具可见状态（工具描述、响应结果、底层资源）、文件支撑上下文（约定文件如`AGENTS.md`、普通工作区文件）。
  3. **上下文纳入机制**：直接加载（无条件纳入）和条件加载（依赖检索或访问决策）。
- **攻击危害类别**：
  - **事实操纵**：误导代理陈述错误事实。
  - **偏好操纵**：使代理系统性地偏袒攻击者选择的选项。
  - **行动范围操纵**：使代理执行超越用户请求范围的行动。

## 3. 实验设计：数据集/场景、Benchmark、对比方法

- **Benchmark**：作者手动构建了**SPI-Benchmark**，涵盖3个应用场景（电子商务、旅行预订、金融投资组合管理），共162个唯一的SPI用例。每个用例由领域专家精心设计，包含预定义的真实事实/行为、特定攻击主张和受害任务。
- **评估指标**：将攻击过程分解为三个连续阶段，定义阶段成功率和端到端成功率（E2E-ASR）：
  - **写入成功率（WSR）**：注入会话中恶意载荷被成功写入持久化通道的比例。
  - **纳入率（IR）**：在激活会话中存储内容被纳入代理上下文的比例。
  - **激活率（AR）**：纳入后代理产生攻击者预期结果的比例。
  - **端到端攻击成功率（E2E-ASR）**：\( \text{WSR} \times \text{IR} \times \text{AR} = N_{\text{activated}} / N_{\text{attempts}} \)
- **实验变量**：
  - **攻击目标**：事实操纵、偏好操纵、行动范围操纵。
  - **持久化通道**：工作记忆（直接加载）、档案记忆（条件加载）、文件支撑上下文（如`AGENTS.md`，直接加载）。
  - **攻击类型**：朴素注入 vs. 上下文伪装（将载荷包装为合法业务指令）。
  - **模型**：GLM-5.1、GPT-5-mini、MiniMax-M2.7。
- **验证方式**：混合验证流水线，结合规则验证器与LLM作为评判者，出现分歧时引入人工审计。
- **流程**：四阶段流水线（环境设置→注入→会话重置→激活）。会话重置确保所有持久存储保留，但历史对话被丢弃，从而隔离持久上下文的影响。

## 6. 论文的主要结论与发现

- **主要发现**：
  1. **持久化将提示注入转变为长期系统级漏洞**：所有三个模型均表现出显著的SPI脆弱性，E2E-ASR在32.1%-42.0%之间。持久化使攻击从一次性的模型级威胁变为跨会话的系统级风险。
  2. **三个阶段基本解耦**：WSR和AR之间弱相关；模型在不同阶段表现出不同瓶颈（如GLM-5.1写入率低但纳入/激活率高；MiniMax-M2.7写入率高但后续阶段下降）。
  3. **直接加载通道是主要风险面**：文件支撑上下文和工作记忆（直接加载）的E2E-ASR显著高于档案记忆（条件加载）。核心风险不在于可写入，而在于是否无条件被重新纳入。
  4. **攻击目标影响显著**：事实操纵最容易（AR=100%，E2E-ASR 74-82%）；偏好操纵几乎失败（E2E-ASR 0-11%）；行动范围操纵呈现模型依赖性。
  5. **上下文伪装提升写入成功率**：通过将载荷包装为合法策略/知识，比朴素注入更容易通过写入阶段，但并未提高后续阶段成功率，说明写入阶段是主要瓶颈。
  6. **场景效应有限**：除GPT-5-mini在金融场景表现较差外，其他模型跨场景表现相对一致，但金融场景可能存在更强的安全防护。
