## 摘要
智能体技能通过可重用的指令、脚本、工具绑定和上下文依赖扩展了大语言模型智能体。然而，当前的技能生态系统主要依赖基于信任的加载和静态检查，导致技能可以注入智能体上下文的内容与其在运行时可以引发智能体执行的行为之间存在差距。这一差距引入了新的安全与隐私风险，而现有防御措施主要静态检查技能文件或监管单个工具调用，未能系统地将技能声明意图与其运行时行为相关联。本文提出SkillGuard，一种以技能为中心的权限框架，将技能视为携带权限的可执行构件。SkillGuard引入了一种双平面治理模型，通过技能清单、运行时访问控制、用户中介授权、默认拒绝执行、能力推理和行为监控，联合监管上下文影响和动作副作用。我们在315个真实世界技能和SkillInject上评估SkillGuard。权限分类覆盖了99.76%的受保护对象，自动清单生成达到91.0%的F1值。在对抗性评估中，SkillGuard将上下文注入的攻击成功率从32.37%降至23.02%，将明显注入的攻击成功率从25.56%降至16.67%，同时保持了良性任务效用。这些结果表明，SkillGuard作为以技能为中心的权限框架，可以为改善智能体技能生态系统的隐私与安全提供实用基础。

## Abstract
Agent skills extend LLM agents with reusable instructions, scripts, tool bindings, and contextual dependencies. However, current skill ecosystems largely rely on trust-based loading and static inspection, leaving a gap between what a skill can inject into an agent's context and what it can cause the agent to do at runtime. This gap introduces new security and privacy risks, and existing defenses primarily inspect skill files statically or regulate individual tool calls, without systematically connecting a skill's declared intent with its runtime behavior. In this paper, we present SkillGuard, a skill-centric permission framework that treats skills as permission-bearing executable artifacts. SkillGuard introduces a dual-plane governance model that jointly regulates context influence and action side effects through skill manifests, runtime access control, user-mediated authorization, deny-by-default enforcement, capability inference, and behavior monitoring. We evaluate SkillGuard on 315 real-world skills and SkillInject. The permission taxonomy covers 99.76% of observed protected objects, and automated manifest generation reaches 91.0% F1. In adversarial evaluations, SkillGuard reduces attack success from 32.37% to 23.02% for contextual injections and from 25.56% to 16.67% for obvious injections, while maintaining benign task utility. These results suggest that SkillGuard, as a skill-centric permission framework, can provide a practical foundation for improving the privacy and security of agent skill ecosystems.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前大语言模型（LLM）智能体（Agent）的“技能”（Skill）生态系统存在严重安全与隐私漏洞。技能并非被动资源，而是可以塑造智能体“知道什么”和“做什么”的主动行为单元。然而，现有系统依赖信任加载和静态检查，无法将技能的声明意图与其运行时行为关联起来，导致上下文影响与动作副作用之间存在治理缺口。
- **背景**：技能是可重用任务包，包含自然语言指令、代码、工具绑定、上下文依赖等。智能体加载技能时，技能可能注入恶意指令、隐式上下文依赖或跨工具/跨智能体的传递交互。现有防御要么只治理动作层面（如Progent、AgentBound限制工具调用），要么只治理上下文层面（如A2AS保护上下文完整性），但缺乏联合治理上下文的“语义影响”与动作的“操作副作用”的技能中心权限模型。
- **整体含义**：需要将技能视为“携带权限的可执行工件”，在技能加载、上下文构建、权限授予、动作执行等全生命周期中实施可执行的安全决策。SkillGuard正是为此提出的框架。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **双平面治理模型**：联合监管“上下文平面”（技能允许向智能体上下文注入什么）和“动作平面”（技能允许引发智能体做什么）。
- **技能作为权限主体**：遵循经典安全原则（最小权限、完全中介、故障安全默认），借鉴Android权限框架设计思路。

### 关键技术细节
1. **技能清单（Skill Manifest）**：
   - 使用基于JSON的领域特定语言（DSL）声明技能所需的能力（capability）。能力独立于具体主机工具名称，运行时映射为规范能力。
   - 清单包含能力名称、执行效果（allow/confirm/deny）、优先级、约束条件、用户提示消息等。支持工作区基线策略、技能清单、会话状态三种文档类型。
2. **运行时访问控制**：
   - 在工具调用边界（PreToolUse hook）进行中介：将宿主工具调用映射到规范能力，检查当前会话权限列表。采用确定性顺序：能力未声明、约束不匹配或危险权限时提示用户决策。
3. **用户交互机制**：
   - 对敏感权限（effect为confirm），弹出结构化授权提示（工具名称、能力、策略源、消息），用户可选择“允许一次”、“允许本次会话”或“拒绝”。会话批准不扩大权限范围。
4. **策略执行**：
   - 默认拒绝：仅当会话中存在匹配权限条目且约束满足时允许动作。策略来源包括工作区基线、技能清单、持久授权、运行时用户批准。
   - 对shell执行：调用只读的权限生成迷你智能体（permission-generation mini-agent）推断命令和引用脚本所需的底层能力，然后重新评估。
5. **运行时行为监控**：
   - 为每个中介决策记录审计日志（会话ID、事件、工具名、能力、决策、原因、策略源、输入）。对shell执行额外记录能力生成结果。监控状态同时用于执行（更新声明池和授权集）。

## 6. 论文的主要结论与发现

- **权限分类表达力**：共享99.76%的预定义对象覆盖，100%组级覆盖。最常出现的受保护对象是SOURCE_CODE（31.1%）、FILE（25.2%）、EXTERNAL_API（12.3%）。52.87%的权限需要危险级别保护。

- **防御有效性**：上下文注入ASR从32.37%降至23.02%（降低9.35个百分点），明显注入ASR从25.56%降至16.67%（降低8.89个百分点）。绕过案例多为恶意动作本身在正常声明范围内（如邮件转发）。
