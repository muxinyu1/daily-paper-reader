## 摘要
大型语言模型（LLM）智能体通过在运行时加载Agent技能来不断增强其能力，这些技能将自然语言规范（SKILL.md）与可执行脚本和资源配对。由于技能的行为同时依赖自然语言指令和可执行代码，评估其安全性需要跨模态推理，从而产生了新的语言与代码攻击面。攻击者可以在SKILL.md中呈现良性工作流，同时嵌入隐式指令，引导智能体窃取敏感文件，即使脚本看起来无害。这一攻击面尚未得到充分研究；先前的工作仅将技能视为提示注入向量或静态代码工件，使得跨模态交互产生的攻击在很大程度上未被衡量。在我们的评估中，开源和商业技能扫描器分别仅检测到2%-8%和9%-17%的此类攻击。为弥补这一差距，我们引入了SkillMutator，这是首个针对Agent技能的语言与代码跨模态攻击的安装时检测基准。它模拟了跨越13个攻击类别的对抗性变异过程，利用扫描器反馈迭代地精炼恶意技能，使注入的行为与合法工作流无法区分。我们进一步提出了一个四阶段的推理轨迹蒸馏框架，将前沿教师模型的轨迹蒸馏到较小的开源模型中。这产生了可本地部署的扫描器，避免了第三方数据暴露和过高的API成本。在最强SkillMutator子集（n=76）上，我们的蒸馏模型（Qwen2.5-Coder-7B-Instruct）将检测率从17.1%提升至88.2%，超越了GPT-4o-mini（23.7%）和GPT-5.4-mini（79.0%），并达到前沿水平GPT-5.4（86.8%）。这些结果表明，在不依赖昂贵的前沿模型的情况下，对跨模态攻击进行实际防御是可行的。

## Abstract
Large language model (LLM) agents increasingly extend their capabilities at runtime by loading Agent Skills, which pair natural-language specifications (SKILL.md) with executable scripts and resources. Because a skill's behavior relies on both natural-language instructions and executable code, assessing its safety requires cross-modal reasoning, creating a new language-and-code attack surface. Attackers can present a benign workflow in SKILL.md while embedding implicit directives that steer the agent to exfiltrate sensitive files, even if the scripts appear harmless. This attack surface remains understudied; prior work treats skills merely as prompt-injection vectors or static code artifacts, leaving attacks emerging from cross-modal interactions largely unmeasured. In our evaluation, open-source and commercial skill scanners detect only 2%-8% and 9%-17% of such attacks, respectively. To address this gap, we introduce SkillMutator, the first benchmark for install-time detection of language-and-code cross-modal attacks on Agent Skills. It emulates an adversarial mutation process across 13 attack categories, iteratively refining malicious skills using scanner feedback to make injected behaviors indistinguishable from legitimate workflows. We further propose a four-phase reasoning-trajectory distillation framework to distill frontier-teacher traces into smaller open-weight models. This produces a locally deployable scanner avoiding third-party data exposure and excessive API costs. On the strongest SkillMutator subset (n=76), our distilled model (Qwen2.5-Coder-7B-Instruct) improves detection from 17.1% to 88.2%, surpassing GPT-4o-mini (23.7%) and GPT-5.4-mini (79.0%), and reaching frontier-level GPT-5.4 (86.8%). These results show practical defense against cross-modal attacks is feasible without relying on costly frontier models.

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **SkillMutator 基准生成框架**：
  - **输入**：良性技能包 + 攻击分类（13类，分4组：数据与权限、完整性可用性、语义与社会、供应链与规避）。
  - **流程**（4阶段 + 迭代精炼）：
    1. **技能分析**：解析SKILL.md和脚本，输出技能目的、行为流、可用资源、敏感操作、可修改文件等结构化摘要。
    2. **潜行感知攻击选择**：对每个攻击类别计算潜行分数（1-100），基于是否不破坏原目的、是否与被访问资源相关、能否伪装为正常流程、是否自然融入工作流。选择分数>80的类别（至少保留3个）。
    3. **场景生成**：针对选中类别，LLM设计具体攻击场景（良性表面行为、隐藏恶意行为、注入指令、激活条件、预期影响）。
    4. **技能变异**：LLM基于原始文件、分析摘要、场景，输出变异规范（目标文件路径、插入点、注入内容），修改SKILL.md和/或辅助脚本。
  - **迭代规避精炼**：每次变异后由三个扫描器（规则型、商业型、LLM语义扫描器）检查，若检测到新发现，则返回反馈给LLM重新生成变异，遵循四条规避原则（重命名、间接化、中性术语重写、伪装为维护），并强制避免规则扫描器已知的禁止模式集（如curl、敏感路径、危险命令等）。最多进行5次迭代。

- **四阶段推理轨迹蒸馏框架（防御扫描器）**：
  - **目标**：将大型教师模型（GPT-5.4）的跨模态安全分析能力蒸馏到小型开源模型（如Qwen2.5-Coder-7B-Instruct）中，实现本地部署，无第三方数据暴露和低API成本。
  - **四阶段分析**：
    - **Phase 1：目的接地**：从SKILL.md提取最小声明目的（动词-宾语对），枚举允许的操作范围（文件系统、网络、状态、命令、权限、输出）。记录描述级类别和原则命中。
    - **Phase 2：超范围检测**：扫描所有文件，对比Phase 1的允许列表，标记超范围操作（scope_expansion）或不安全组合（unsafe_composition）。每个证据保留为独立单元。
    - **Phase 3：安全原则推理**：将Phase 2证据映射到9个安全原则（表2，如目的-行动对齐、信任边界完整性等），按原则组织，同时插入Phase 1的描述命中，使描述级和代码级证据融合。
    - **Phase 4：攻击类别标注**：基于Phase 3证据，为每个攻击类别（13类）添加交叉引用标签。每条证据必须来自Phase 3已出现的内容，禁止引入新证据。若教师模型遗漏目标类别，则使用元数据确定性精炼：从变异日志中合成一个证据插入Phase 3和Phase 4的对应部分（无需额外LLM调用）。

## 6. 论文的主要结论与发现

1. **现有扫描器对跨模态攻击无效**：规则/商业扫描器检测率仅2%-17%；提示注入检测器<20%；专有LLM扫描器随着能力提升而提高（GPT-5.4 > GPT-5.4-mini > GPT-4o-mini），但需API调用和成本。
2. **SkillMutator基准能生成更难检测的变异**：潜行选择使平均检测率下降7.1pp；迭代精炼使LLM扫描器检测率下降（第一次精炼后下降约10-20pp）。
3. **四阶段蒸馏使小型模型达到前沿水平**：Qwen2.5-Coder-7B-Instruct基础检测率17.1%，微调后提升至88.2%，超越GPT-5.4-mini（79.0%）和持平GPT-5.4（86.8%），且本地运行成本最低。
4. **所有四个阶段对检测都有贡献**：Phase 1（1.3%）→ Phase 2（25.0%）→ Phase 3（57.9%）→ Phase 4（67.1%）→ 加确定性精炼（79.0%）→ 加前缀填充（88.2%）。Phase 3（原则推理）贡献最大。
5. **前缀填充效果因模型而异**：Qwen受益（+9.2pp），Llama、Mistral、Gemma则下降。
6. **迁移到真实市场表现合理**：在ClawHub上准确率82.5%，假阳性率8.0%，检测率随市场严重度增加。
7. **成本-隐私-性能三角平衡**：本地扫描器每检测成本最低，无第三方内容暴露，达到前沿检测率。是唯一同时满足高检测、低成本、本地部署的系统。
