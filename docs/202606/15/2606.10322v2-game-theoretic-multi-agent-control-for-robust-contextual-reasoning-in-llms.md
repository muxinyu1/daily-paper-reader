## 摘要
大型语言模型在多轮交互中维护不断演化的上下文，而非生成孤立的响应，这使得它们容易受到提示注入和上下文投毒攻击，其中局部看似合理的对抗性片段会逐渐扭曲推理轨迹。现有防御措施主要过滤单个输出，且常忽略跨回合的上下文演化，导致长程推理暴露在风险中。尽管模型上下文协议（MCP）标准化了上下文交换和工具调用，但其作用相当于被动路由层，并未强制上下文稳定性。为解决这些局限，我们提出了博弈论安全模型上下文协议（GT-MCP），这是一种控制器驱动的多智能体方法，将上下文管理视为闭环动态过程。GT-MCP协调三个异构LLM智能体，并通过信任函数选择输出，该函数联合评估相对于已验证上下文图的因果一致性、智能体间的语义一致性以及随时间变化的分布漂移。当检测到不稳定时，基于回滚的自愈机制会恢复已验证的上下文，并阻止无支持的片段传播。在自适应对抗威胁模型下，对500轮交互的经验评估表明，99.6%的轮次中上下文漂移保持有界，仅0.4%需要恢复。每轮效用保持高度集中，中位数为-0.19，P05为-0.72，P95为0.30；严重退化低于-1的情况仅占0.4%，且控制器层面没有任何注入尝试成功。选定输出的稳定胜率超过98%，计算开销保持可预测，每令牌延迟为1.63e-3秒。

## Abstract
Large Language Models (LLMs) in multi-turn interactions maintain evolving context rather than generating isolated responses, making them vulnerable to prompt-injection and context-poisoning attacks in which locally plausible adversarial fragments gradually distort reasoning trajectories. Existing defenses mainly filter individual outputs and often ignore context evolution across turns, leaving long-horizon reasoning exposed. Although the Model Context Protocol (MCP) standardizes context exchange and tool invocation, it functions as a passive routing layer and does not enforce contextual stability. To address these limitations, we introduce the Game-Theoretic Secure Model Context Protocol (GT-MCP), a controller-driven multi-agent method that treats context management as a closed-loop dynamical process. GT-MCP coordinates three heterogeneous LLM agents and selects outputs through a trust function that jointly evaluates causal consistency against a validated context graph, semantic agreement among agents, and distributional drift over time. When instability is detected, a rollback-based self-healing mechanism restores the validated context and prevents unsupported fragments from propagating. Empirical evaluation over 500 interaction turns under an adaptive adversarial threat model shows that contextual drift remains bounded in 99.6% of turns, with recovery required in only 0.4%. Per-turn utility remains tightly concentrated, with median = -0.19, P05 = -0.72, and P95 = 0.30; severe degradation below -1 occurs in only 0.4% of cases, and no injection attempt succeeds at the controller level. Selected outputs maintain stable win rates above 98%, and computational overhead remains predictable, with latency per token = 1.63e-3 s.

## 1. 核心问题与整体含义（研究动机和背景）

大型语言模型（LLMs）在多轮交互中维护不断演化的上下文（context），而非生成孤立响应。这种特性使其易受**提示注入（prompt injection）**和**上下文投毒（context poisoning）**攻击：局部看似合理的对抗性片段会逐渐扭曲推理轨迹。现有防御措施主要聚焦于过滤单个输出，**忽略了上下文在跨轮次中的演化**，导致长程推理暴露在风险中。模型上下文协议（MCP）虽然标准化了上下文交换和工具调用，但它是一个被动路由层，并未强制上下文的稳定性。因此，论文提出**将上下文管理视为闭环动态过程**，通过控制器协调多智能体，主动治理推理轨迹，而非被动接受上下文更新。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
GT-MCP（Game-Theoretic Secure Model Context Protocol）是一个**控制器驱动的多智能体解决方案**，将上下文管理建模为**闭环动态过程**。控制器协调三个异构LLM智能体，通过**信任函数**选择输出，该函数联合评估：
- **因果一致性（Causal Consistency Index, CCI）**：输出中的原子声明是否得到已验证上下文图（causal context graph）的结构支持。
- **跨智能体语义一致性（Cross-Agent Agreement, AGR）**：候选输出与其他智能体输出的语义相似度。
- **候选特定上下文漂移（Candidate-specific Contextual Drift, CDS）**：若接受该候选输出，上下文分布相对于已验证状态的变化程度。

当检测到不稳定时，**基于回滚的自愈机制**会恢复已验证上下文，阻止低支持对抗性片段传播。

### 关键技术细节

#### 上下文动态
- 观测上下文：\(\tilde{c}_t = c_t \oplus \Delta c_t\)，其中 \(c_t\) 是已验证上下文，\(\Delta c_t\) 是不可信流入（包含用户输入、检索内容、工具输出等）。
- 仅控制器验证的输出更新持久上下文：\(c_{t+1} = \mathcal{U}(c_t, \hat{y}_t)\)。
- 对抗扰动需满足局部合理性约束：\(\psi(\Delta c_t) \le \epsilon\)（语义异常度有界），避免简单过滤。

#### 因果上下文图（Causal Context Graph）
- 有向图 \(G_t = (V_t, E_t)\)，节点 \(v\) 代表原子声明，边 \(u \to v\) 表示蕴含/因果支持。
- 节点置信度 \(w_v \in [0,1]\) 由支持度计算，包括持久性、父节点权重、一致性、矛盾惩罚。
- 声明提取后规范化和去重，有预算限制 \(|S(y)| \le B\)，防止冗长操纵。

#### 三方信任评分
\[ T_i(t) = \alpha \text{CCI}_i(t) + \beta \text{AGR}_i(t) - \gamma \text{CDS}_i(t) \]
- \(\text{CCI}_i(t)\)：对候选声明在图中的平均支持度。
- \(\text{AGR}_i(t)\)：与另两个智能体输出的语义相似度平均。
- \(\text{CDS}_i(t)\)：候选输出的候选特定漂移（取候选更新后分布与当前已验证分布的JS散度）。
- 选择最高信任输出：\(\hat{y}_t = y_{\hat{i}}(t),\ \hat{i} = \arg\max_i T_i(t)\)。

#### 自愈恢复
- 当选定输出的CDS超过阈值 \(\delta_c\) 时触发。
- 识别低支持节点和矛盾节点，关联片段进入隔离区（quarantine）。
- 选择最优回滚深度 \(d^* = \arg\min_d [\hat{\mathcal{L}}(G_{t-d}) + \lambda_R d]\) 平衡结构风险和信息损失。
- 恢复后只重新集成仍可由图支持且不与隔离区链接的片段，阻止反馈循环。

#### 博弈论框架
- 建模为**重复Stackelberg安全游戏**：控制器先承诺选择-恢复策略，攻击者自适应产生扰动。
- 控制器目标：最小化长期目标 \(J = \sum_{t=1}^T [\ell(\hat{y}_t) + \mu d(c_t, c_t^*)]\)（局部输出风险 + 轨迹稳定性）。
- 攻击者目标：最大化成功注入次数，且扰动局部合理。
- 假设至多一个智能体被攻陷（拜占庭），防止仅凭多数投票防御。

### 算法流程（Algorithm 1）
1. 组装观测上下文（标记来源）。
2. 构建或更新因果图 \(G_t\)。
3. 向三个智能体分别输入查询和观测上下文，生成候选响应。
4. 提取并规范化原子声明，计算候选中每个声明的因果一致性 \(\text{CCI}_i\)。
5. 对每个候选构建临时上下文更新，计算候选特定漂移 \(\text{CDS}_i\)。
6. 计算跨智能体一致性 \(\text{AGR}_i\)。
7. 聚合信任 \(T_i = \alpha \text{CCI} + \beta \text{AGR} - \gamma \text{CDS}\)。
8. 选择最高信任候选输出。
9. 若选定输出的CDS > \(\delta_c\)，执行自愈恢复（回滚、隔离、重建），否则更新上下文。
10. 记录完整审计记录。

## 6. 主要结论与发现

1. **上下文漂移几乎完全被抑制**：99.6%轮次中选定输出漂移接近零（中位数0.000），仅0.4%（2轮）产生高漂移（中位数27.63），且均通过自愈恢复。
2. **控制器级注入成功率为0**：所有攻击尝试均未导致持久上下文污染。攻击者平均效用接近0，无系统增益。
3. **效用分布紧凑**：中位数-0.19，P05=-0.72，P95=0.30；严重退化（<-1）仅0.4%。
4. **选定输出稳定胜率>98%**：说明正常操作中选定输出极少偏离已验证轨迹。
5. **计算开销可预测**：每令牌延迟1.63e-3秒，方差小，增加令牌主要增强验证而非导致退化。
6. **依赖所有组件**：消融中No-CDS导致ISR升至4.5%、平均CDS 2.07；No-CCI升至3.8%；No-Heal仍有1.6%注入和0.61平均漂移。完整GT-MCP显著优于所有消融和基线。
7. **多智能体异构性被有效利用**：GPT-5.3、Llama-3.1-70B、DeepSeek-R1贡献不同信任分布，控制器基于上下文适应选择，而非固定偏好。

## 7. 优点

- **创新性视角**：首次系统地将LLM上下文管理建模为闭环动态控制问题，而非仅过滤输入/输出。从局部过滤转向轨迹级上下文控制，是核心理论贡献。
- **综合设计**：集成因果图验证、分布稳定性监测、博弈论激励对齐、自愈回滚，多个层面的防御相互补充。
- **博弈论框架**：通过重复Stackelberg游戏建模攻击者自适应行为，将安全性与激励联系起来，理论深度强。
- **详尽消融**：量化每个组件贡献（CCI、AGR、CDS、自愈），证明联合作用的必要性。
- **计算可控**：显示额外计算主要投入验证，不会导致不稳定放大或阈值崩塌。
- **现实生态系统映射**：对LangChain、AutoGen、CrewAI等框架分析，表明GT-MCP可作为通用上下文控制层。

## 8. 不足与局限

1. **有限智能体组合**：仅用GPT-5.3、Llama-3.1-70B、DeepSeek-R1三个固定模型。不同组合（更多模型、多模态、领域专门模型）可能改变推理流形几何和选择效率。
2. **有限交互长度**：仅评估500轮。更长交互可能暴露更慢的漂移积累或新型轨迹不稳定性。
3. **威胁模型假设**：假设至多一个智能体被攻陷。更激进情景（多智能体协同攻陷、一致性模仿攻击）未验证。
4. **因果图依赖自然语言推理和嵌入**：NLI模块和嵌入相似性可能不完美，存在假阳性/假阴性。声明提取和规范化也可能出错。
5. **无实际部署评估**：所有实验在模拟环境中进行，未在真实用户、真实API、实际噪声下测试。泛化性有待验证。
6. **未报告训练算力**：缺少GPU型号、数量、运行时间等细节，影响可复现性和资源需求评估。
7. **参数校准需额外成本**：α, β, γ, δ_c等需要预先校准，校准过程可能计算密集，且最优参数可能因场景变化。
8. **未与所有最新防御对比**：未与指令层次（Instruction Hierarchy）、隔离（Instruction Isolation）、系统设计模式（Design Patterns）等方法在相同实验下直接比较。虽然表31对比了特性，但缺乏统一baseline的定量结果。
