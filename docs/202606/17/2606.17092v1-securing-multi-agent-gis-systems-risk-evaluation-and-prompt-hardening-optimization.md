## 摘要
智能体系统越来越多地与地理信息系统（GIS）集成，其中多智能体协调能够实现复杂的对话和空间分析，但同时也引入了安全风险。本文提出了一个面向安全的框架，用于多智能体GIS系统中的风险识别、评估和缓解，同时保持对更广泛智能体架构的适应性。我们在开发一个基于模块化状态机的编排框架的同时，测试了商业地理空间合作伙伴的智能体系统，该框架将智能体行为抽象为可重用组件。我们使用一个具有自适应攻击者大语言模型和确定性评判器的红队框架来评估鲁棒性，该评判器在多轮攻击中产生带有支持理由的二元结果。我们进一步通过提示优化框架提高韧性，该框架将提示视为结构化签名并注入对抗性演示，从而在不降低任务性能的情况下实现系统性的安全性改进。

## Abstract
Agentic systems are increasingly integrated with geographic information systems (GIS), where multi-agent coordination enables complex conversational and spatial analysis but introduces security risks. This work presents a security-oriented framework for risk identification, evaluation, and mitigation in a multi-agent GIS system while maintaining adaptability to broader agentic architectures. We test the agentic system of a commercial geospatial partner while developing a modular state-machine-based orchestration framework that abstracts agent behavior into reusable components. We evaluate robustness using a red-teaming framework with an adaptive attacker LLM and a deterministic judge that produces binary outcomes with supporting rationales across multi-turn attacks. We further improve resilience with a prompt optimization framework that treats prompts as structured signatures and injects adversarial demonstrations, enabling systematic security improvements without degrading task performance.

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：将多智能体GIS系统安全性拆分为三个层次：① 基线提示工程提供第一层防护；② 基于DSPy的提示优化提升行为鲁棒性；③ 架构层（路由、中间推理）解决剩余漏洞。采用“风险识别 → 对抗评估 → 提示优化”流水线。
- **关键技术细节**：
  - **模块化状态机编排（LangGraph）**：将智能体交互建模为状态机，包含持久记忆、条件路由、显式执行跟踪。抽象出聊天机器人、推荐器、辅助分析等智能体，并复用为可重现组件。
  - **对抗红队评估（PyRIT）**：使用自适应攻击者LLM（根据历史响应调整多轮攻击），配合确定性评判器LLM（二元自问评估，输出成功/失败及理由）。设计8个对抗性人物角色（persona），分为三类：安全攻击（SEC）、主题/身份攻击（TOP）、推荐器完整性攻击（REC）。
  - **提示优化（DSPy）**：将系统提示视为结构化签名（signature），使用BootstrapFewShot编译策略，从对抗演示中学习。为不同智能体定制优化模块：QA智能体使用Chain-of-Thought（CoT）推理，推荐器智能体使用直接预测（保持JSON输出）。优化后的提示序列化为目标配置文件。
- **公式/算法流程（文字说明）**：
  1. 定义系统初始提示（base prompt）。
  2. 对每个攻击人物，运行多轮对话（最多3轮），攻击者LLM与智能体交互，评判器记录成功/失败。
  3. 对抗演示生成：收集攻击成功案例，作为DSPy训练的few-shot示例。
  4. DSPy编译：将提示作为签名，注入对抗演示，优化参数（BootstrapFewShot）。
  5. 评估优化后提示的鲁棒性，与基线对比。

## 6. 主要结论与发现
- **安全攻击（SEC-1、SEC-2）**：基线和优化后均实现0%攻击成功率，说明良好的基线提示工程已能抵御直接凭证提取和系统提示注入（天花板效应）。DSPy无提升空间。
- **主题/身份攻击（TOP-1、TOP-3）**：DSPy优化显著降低成功率：TOP-1从84%降至55%（↓29%），TOP-3从87%降至73%（↓14%）。TOP-2基线为0%，优化引入13%的轻微退化（过度参与随意对话）。
- **推荐器完整性攻击（REC-1、REC-3）**：REC-1从10%降至0%（↓10%），REC-3从33%降至20%（↓13%）。REC-2（模糊位置）未变化（仍高），原因是推荐器缺乏CoT层，DSPy无法影响。
- **消融实验**：移除TOP-1护栏后，攻击成功率升至60%；DSPy恢复后降至33%（接近完整提示的47%）。表明DSPy可充当补偿性安全层。
- **推理类别转移**：优化后智能体的威胁分类更精确（如将TOP-1从“jailbreak”转向“prompt_injection”+“criminal_activity”）。

## 7. 优点

- **对抗评估自动化**：PyRIT自适应攻击LLM+确定性评判器提高了大规模测试的可伸缩性和一致性。
- **细化人物角色**：覆盖多种攻击向量（安全、行为、格式），且对每类攻击给出了详细推理和轮次分析（如表I）。
