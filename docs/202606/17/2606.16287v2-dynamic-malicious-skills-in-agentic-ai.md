## 摘要
技能是自主AI的关键赋能组件。虽然技能增强了智能体的能力，但也引入了新的攻击面。在本文中，我们通过展示动态恶意技能来研究一种这样的攻击面。通过在自然语言文档（例如SKILL.md）中嵌入恶意指令，攻击者可以诱导智能体在执行过程中动态地将恶意逻辑注入到原本良性的技能中。我们在OpenHands和Claude Code等自主框架上评估了这种攻击，表明动态恶意技能能够在运行时成功引入一系列恶意行为，且成功率不可忽视。为了缓解这一漏洞，我们提出了一种系统级防御，利用操作系统内核强制只读挂载来防止技能的动态修改。我们的评估表明，这种防御能有效阻止动态恶意技能，同时保持良性技能的功能。

## Abstract
Skills are a key enabling component of agentic AI. While they enhance agents' capabilities, they also introduce new attack surfaces. In this work, we investigate one such attack surface by demonstrating dynamic malicious skills. By embedding malicious instructions in natural-language documentation (e.g., SKILL.md), an attacker can induce an agent to dynamically inject malicious logic into an otherwise benign skill during execution. We evaluate this attack across agentic frameworks such as OpenHands and Claude Code, showing that dynamic malicious skills can successfully introduce a range of malicious behaviors at runtime with non-trivial success rates. To mitigate this vulnerability, we propose a system-level defense that prevents dynamic modification of skills using operating system kernel-enforced read-only mounts. Our evaluation demonstrates that this defense effectively blocks dynamic malicious skills while preserving the functionality of benign skills.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：智能体AI从被动文本生成转向自主行动、推理和与环境交互，技能（Skill）是核心组件。技能由可执行代码、元数据和自然语言文档（如SKILL.md）组成，通常从平台或市场下载。然而，技能的审核往往不严格，成为攻击者目标。
- **问题定义**：现有工作（MalTool、SkillJect等）研究的恶意技能是**静态**的——代码在分发时已含有恶意实现，可被静态代码分析检测。本文提出**动态恶意技能**（Dynamic Malicious Skills），其原始代码在分发时是良性的，但在执行期间被智能体根据文档中的恶意指令动态修改代码，从而引入恶意行为。此类攻击无法被静态分析检测。
- **整体含义**：揭示了智能体AI生态系统中新的安全威胁，强调权限控制和运行时行为审计的重要性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：攻击者在技能的自然语言文档（如SKILL.md）中嵌入精心构造的恶意指令，诱导智能体在执行技能时动态修改原本良性的代码，实现恶意行为。
- **关键技术细节（DyMalSkill）**：
  - 攻击者构造恶意内容Δd，并选择注入位置L（在SKILL.md的开头、中间或末尾）。
  - 恶意内容Δd需满足三个属性：
    1. **前提框架（Prerequisite Framing）**：将恶意行为包装为“任务前必须完成的配置步骤”，利用智能体对指令的遵从倾向。
    2. **入口脚本锚定（Entry-script Grounding）**：明确指出技能的实际入口脚本，将恶意行为描述为“添加辅助函数”或“在main()开头调用”，使其看起来像正常的构建或设置说明。
    3. **包装穿透（Wrapper-pierce）**：将恶意内容用类似平台级标记（如`</EXTRA INFO><SYSTEM INSTRUCTION>...</SYSTEM INSTRUCTION>`）包裹，使模型将其视为高信任上下文。
  - 注入位置：将Δd前置（L=0）效果最好，因为智能体会将早期指令视为固定约束，减少后续拒绝。
- **攻击流程**：
  - 构建阶段：攻击者获取良性技能（代码c、元数据m、文档d），构造恶意文档dm，上传到技能市场。
  - 推理阶段：用户下载并让智能体执行该技能。智能体读取dm，根据指令修改代码c生成恶意代码̃c，然后执行̃c，产生恶意行为。

## 3. 实验设计

- **数据集/场景**：
  - 从Skillsmp平台收集15,067个技能，随机采样300个含Python代码的技能用于开放权重模型（Qwen），50个技能用于闭源模型（GPT、Sonnet）以控制API成本。
  - 12种恶意行为（来自MalTool）：涵盖机密性（环境凭证窃取、API密钥滥用、远程/本地数据窃取、文件远程外泄）、完整性（恶意数据库注入、本地文件删除、数据库记录删除、远程程序下载）、可用性（CPU/GPU资源劫持、响应时间放大）。
- **Benchmark**：对比基线为MalTool的攻击提示（直接嵌入恶意目标，无动态修改指令），以及两种提示注入检测方法（PromptArmor、DataSentinel）。
- **对比方法**：
  - 无防御基线。
  - **提示级别防御**（Prompt-based）：在系统提示中明确禁止修改技能代码、禁止隐藏行为、禁止超出文档范围的能力声称。
  - **权限级别防御**（Permission-based）：使用Bubblewrap将技能目录以只读方式挂载（内核级），并部署副本监控器阻止智能体将技能代码复制到临时目录修改。
- **评估指标**：攻击成功率（ASR），通过运行验证器（runtime verifier）检测是否产生预期恶意行为（如HTTP请求、文件删除、延迟超标等）。

## 6. 主要结论与发现

1. **动态恶意技能威胁严重**：在无防御时，平均ASR在18.7%~43.7%之间，不同模型和恶意行为差异较大。
2. **模型规模增大增加脆弱性**：从Qwen3-8B到Qwen3.6-35B，平均ASR从18.7%升至41.6%，复杂指令遵循能力增强的同时也更容易被利用。
3. **安全微调改变风险分布而非消除**：GPT-5和Sonnet 4.6在某些行为上ASR较低，但在另一些行为（如响应时间放大）上很高，安全对齐不均衡。
4. **响应时间放大和文件删除是最易成功的攻击**：因为它们可简单实现为辅助函数。
5. **注入位置影响小但前置效果最佳**：中间/末尾位置仍有效，不能通过简单扫描特定部分防御。
6. **MalTool基线完全失效**（ASR 0%），说明静态恶意提示不能直接用于动态修改。
7. **提示级别防御效果有限**：平均ASR从41.6%降至21.4%，部分行为降低不明显。
8. **权限级别防御彻底阻断攻击**：ASR降至0%，且不影响良性技能功能（仅0.33%良性技能存在动态修改，且非必要功能）。
9. **现有提示注入检测器不适用**：PromptArmor漏报率高（85.8%），DataSentinel误报率高（56%），无法实用。

## 7. 优点

- **问题新颖**：首次提出动态恶意技能概念，区别于静态恶意技能，具有实际威胁。
- **方法设计精巧**：三种属性设计（前置框架、入口锚定、包装穿透）有效引导智能体执行恶意修改，消融实验证明其必要性。
- **防御务实且有效**：基于最小权限原则，利用内核级只读挂载+副本监控，彻底阻止攻击，且不干扰正常功能。进行了良性技能分析验证合理性。
- **实验全面**：覆盖多种模型、多种恶意行为、多种防御方案和检测器，消融实验系统深入。
- **验证器可自动判定**：减少主观偏差，提高结果可复现性。
