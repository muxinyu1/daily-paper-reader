## 摘要
使用工具的大型语言模型（LLM）智能体面临两种不同的安全失效：未经授权的外部操作以及在最终输出检查干预之前运行环境中敏感明文暴露。现有防御通常只保护一个边界，即规划器/运行时或动作接收端，因此它们本身无法同时保护这两个方面。我们提出SecureClaw，一种双边界架构，它将授权置于效果接收端，将明文限制置于读取边界。敏感读取通过一个可信网关，该网关用不透明句柄替换原始值，并且在评估的部署中，使用有界摘要作为显式的解分类接口。改变外部状态的写入遵循PREVIEW→COMMIT协议，其中只有可信执行者才能提交由策略授权的确切规范请求。运行时仍然可以基于摘要和符号引用进行规划，但不能直接解引用秘密或执行副作用。在AgentDojo、AgentLeak和Agent Security Bench（ASB）上，SecureClaw是我们在通用测试工具中评估的唯一一种防御，它同时保留了可用任务效用，并在ASB上实现了0%的攻击成功率（ASR），在AgentDojo上实现了0.64%的ASR，在AgentLeak的攻击奇偶测试道上实现了3.23%的整体泄漏（该测试衡量最终输出和内部中继泄漏）。

## Abstract
Tool-using large language model (LLM) agents face two distinct security failures: unauthorized external actions and exposure of sensitive plaintext inside the runtime before any final output check can intervene. Existing defenses usually protect one boundary, either the planner/runtime or the action sink, and therefore do not by themselves secure both surfaces. We present SecureClaw, a dual-boundary architecture that places authorization at the effect sink and plaintext confinement at the read boundary. Sensitive reads pass through a trusted gateway that replaces raw values with opaque handles and, in the evaluated deployment, bounded summaries as an explicit declassification interface. Writes that change external state follow a PREVIEW$\rightarrow$COMMIT protocol in which only a trusted executor may commit the exact canonical request authorized by policy. The runtime can still plan over summaries and symbolic references, but cannot directly dereference secrets or perform side effects. Across AgentDojo, AgentLeak, and Agent Security Bench (ASB), SecureClaw is the only defense we evaluate in a common harness that simultaneously retains usable task utility and achieves 0\% attack success rate (ASR) on ASB, 0.64\% ASR on AgentDojo, and 3.23\% overall leak on AgentLeak's attacked parity lane, which measures final-output and internal-relay leakage.

# 论文总结：SecureClaw：夺回LLM智能体的控制权

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：工具型大型语言模型（LLM）智能体面临两种截然不同的安全失效：
  - **外部动作未授权**：攻击者通过提示注入等方式操控智能体执行未经授权的副作用（如发送邮件、修改记录）。
  - **运行时敏感明文泄露**：敏感数据最终输出前以明文形式暴露在运行环境中，可能通过内部通道（如智能体间消息、共享内存）泄漏。
- **现有缺陷**：现有防御方案通常只保护单一边界（规划/运行时或动作接收端），无法同时防范上述两类威胁。例如，仅拦截动作无法阻止运行时已读取的秘密通过内部通道泄露；仅隔离文本无法阻止未授权动作执行。
- **研究目标**：提出一种同时防护效果表面（effect surface）和明文表面（plaintext surface）的通用架构。

## 2. 方法论：核心思想与关键技术细节
- **核心思想**：将安全控制分离为两个边界：
  - **读取边界（Read Boundary）**：由可信网关中介敏感数据读取，用不透明句柄（opaque handle）和有界摘要替换原始明文，防止运行时直接接触秘密。
  - **写入边界（Write Boundary）**：所有可能改变外部状态的操作通过**PREVIEW→COMMIT协议**，只有可信执行者（Executor）才能提交策略授权的确切请求。
- **关键技术细节**：
  - **读取路径**：可信网关存储敏感明文，返回一个高熵句柄（如`hid = HMAC-SHA256(k_handle, nonce||session||object)`）以及由**确定性、模式感知的有界摘要算子**生成的有界摘要（项目上限8，字符上限512）。运行时只能携带句柄，不可直接解析；句柄仅在可信执行者处解引用。
  - **写入路径协议（Protocol 1 PREVIEW→COMMIT）**：
    1. 运行时提交动作意图和输入（`intent, inputs`）。
    2. 网关附加经过身份验证的调用者、会话和上下文，形成请求`ρ`。
    3. 网关对`ρ`进行规范化（`Canon(ρ)`），并计算绑定摘要`h ← HMAC-SHA256(k_bind, Canon(ρ))`。
    4. 策略引擎审核`h`，返回授权工件。
    5. 执行者独立重算`h`，验证授权工件、新鲜度、防重放、确认令牌等。
    6. 仅当所有检查通过，执行者解引用必要句柄并提交外部效果；否则安全关闭。
  - **拒绝感知恢复（Deny-aware Recovery）**：当提交被拒绝时，返回结构化拒绝（包含动作类别、原因码和安全下一步提示，如“转为草稿待确认”），不泄露新保护的明文内容，保持流程可用性。
- **形式化保证**：论文给出定理1（请求绑定授权完整性）、定理2（仅句柄情况下运行时保密性）、定理3（有界摘要接口的量化暴露），并推导在分布式策略评估下的隐私保持。

## 6. 主要结论与发现
- SecureClaw是唯一同时在三个基准上实现：

  - AgentDojo上0.64% ASR（4/629），攻击效用56.6%，良性效用60.8%。

- **系统化设计**：基于可信执行者与句柄的架构，形式化理论保证（请求绑定、句柄隔离、有界摘要量化）。
- **实验严谨**：多基准、多通道、消融、绕过、恢复、延迟等全面评估；统计显著性检验；跨模型检查；稳定性复现。

- **安全与可用性权衡**：在保持近乎零攻击成功率的同时维持高任务效用（88.9% ASB）。

## 8. 不足与局限
- **信任假设**：依赖网关、句柄存储、策略引擎和执行者等可信组件；如果这些组件被攻破，核心保证失效。论文明确列出各类妥协后果。
- **残留失败**：
  - 效果表面：策略允许但错误的动作（如发错收件人）仍可能执行。
  - 明文表面：有界摘要本身可能泄露区分性信息（实验测量平均∆=0.287），导致C1泄露。

- **实验覆盖**：仅针对单一模型主要评估；跨模型检查样本量小；基准仅覆盖三类攻击模式（提示注入、内部泄露、工具选择），未包括供应链攻击、侧信道等。
